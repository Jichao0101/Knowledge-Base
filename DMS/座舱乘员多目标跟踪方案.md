# 1 目标

实现对驾驶员/乘员的**躯干、头部、双手**进行稳定跟踪，满足以下要求：

- 每个目标具有稳定且固定的 `track id`
- 在短时漏检、局部遮挡、检测抖动场景下保持较好的 ID 连续性
- 对双手能够区分**左手**与**右手**
- 支持驾驶员 / 副驾 / 后排乘员等空间属性的长期稳定判定
- 为疲劳、分心、脱手识别提供稳定输入

---

# 2 总体思路

系统采用“**检测 + 卡尔曼滤波预测 + 匈牙利匹配 + 生命周期管理**”的多目标跟踪框架。

## 2.1 层级设计

整体分为三个层级：

1. **躯干跟踪**  
   负责建立乘员级主体轨迹，作为头部和双手跟踪的主锚点与启动入口。

2. **头部跟踪**  
   在躯干轨迹稳定后启动，优先利用躯干空间约束进行局部匹配和更新；启动后，其轨迹存在不再严格依赖躯干轨迹持续存在。

3. **手部跟踪**  
   在躯干轨迹稳定后启动，为每个乘员分别维护左手和右手轨迹。由于手部姿态变化频繁，采用匀加速运动模型，并在更新时同时更新位置与尺度。

各类轨迹均采用统一的短期状态与长期状态管理机制：

- **短期状态**：解决逐帧匹配、命中、丢失、轨迹延续问题
- **长期状态**：解决轨迹是否稳定输出、人员属性是否可信的问题

## 2.2 基本流程

每一帧的处理流程如下：

1. 读取当前帧检测结果，得到躯干、头部、手部候选框
2. 对上一帧已存在轨迹执行卡尔曼滤波预测，得到当前时刻预测状态和预测框
3. 基于预测框与当前检测框计算匹配损失，构建扩展损失矩阵
4. 通过匈牙利算法进行最优一对一匹配
5. 对匹配成功的轨迹执行卡尔曼滤波更新步，同时更新 `hitCount`
6. 对未匹配检测新建轨迹
7. 对未匹配轨迹更新 `hitCount` 与 `missCount`
8. 根据 `hitCount` 和 `missCount` 更新轨迹生命周期
9. 对达到稳定条件的轨迹输出结果，并更新驾驶员 / 副驾 / 后排、头部唯一、左右手唯一等长期属性

可概括为：

`上一帧稳定轨迹 -> 预测当前状态 -> 与当前检测做最优关联 -> 更新轨迹状态 -> 输出稳定目标`

# 3 轨迹层级设计

## 3.1 躯干跟踪

躯干轨迹是系统的主锚点，负责：

- 建立乘员级身份
- 保持跨帧 ID 稳定
- 提供头部与手部的空间搜索约束
- 提供驾驶员 / 副驾 / 后排等空间属性判定的基础

### 3.1.1 初始化

在首帧或无历史轨迹时，将检测到的躯干框初始化为轨迹：

$\mathcal{D}^{body}_0 = \{d_1, d_2, \dots, d_m\}$，$\mathcal{T}^{body}_0 = \{t_1, t_2, \dots, t_m\}$。

每个检测分配唯一 `trackId`，并初始化命中计数、丢失计数及属性统计计数。

### 3.1.2 状态建模

躯干目标采用位置与速度联合建模。若以框中心点和框尺度作为观测，则状态可表示为  
$\mathbf{x}_t^{body} = [c_x, c_y, w, h, v_x, v_y]^T$。

其中：

- $c_x, c_y$ 表示框中心位置
- $w, h$ 表示框宽高
- $v_x, v_y$ 表示中心点速度

观测向量为 $\mathbf{z}_t^{body} = [c_x, c_y, w, h]^T$。

### 3.1.3 状态预测

对上一时刻更新后的轨迹状态执行卡尔曼滤波预测步，得到当前时刻预测状态：

$\hat{\mathbf{x}}_{t|t-1}^{body} = \mathbf{F}_{body}\mathbf{x}_{t-1|t-1}^{body}$

协方差预测为：

$\hat{\mathbf{P}}_{t|t-1}^{body} = \mathbf{F}_{body}\mathbf{P}_{t-1|t-1}^{body}\mathbf{F}_{body}^T + \mathbf{Q}_{body}$

其中：

- $\mathbf{F}_{body}$ 为躯干状态转移矩阵
- $\mathbf{Q}_{body}$ 为躯干过程噪声协方差矩阵

再由预测状态得到预测框：$Track^{body}_{pred,t} = f(\hat{\mathbf{x}}_{t|t-1}^{body})$。

### 3.1.4 数据关联

将当前检测与预测轨迹构造损失矩阵，通过匈牙利算法进行最优一对一匹配。

### 3.1.5 状态更新

- 若检测匹配到已有轨迹，则执行卡尔曼滤波更新步，并更新轨迹的位置状态
- 若检测未匹配到已有轨迹，则新建轨迹
- 若轨迹未匹配到检测，则更新命中计数与丢失计数

对于匹配成功的轨迹，设检测观测为 $\mathbf{z}_t^{body}$，则更新步为：

$\mathbf{K}_t^{body} = \hat{\mathbf{P}}_{t|t-1}^{body}\mathbf{H}_{body}^T(\mathbf{H}_{body}\hat{\mathbf{P}}_{t|t-1}^{body}\mathbf{H}_{body}^T + \mathbf{R}_{body})^{-1}$

$\mathbf{x}_{t|t}^{body} = \hat{\mathbf{x}}_{t|t-1}^{body} + \mathbf{K}_t^{body}(\mathbf{z}_t^{body} - \mathbf{H}_{body}\hat{\mathbf{x}}_{t|t-1}^{body})$

$\mathbf{P}_{t|t}^{body} = (\mathbf{I} - \mathbf{K}_t^{body}\mathbf{H}_{body})\hat{\mathbf{P}}_{t|t-1}^{body}$

其中：

- $\mathbf{H}_{body}$ 为观测矩阵
- $\mathbf{R}_{body}$ 为观测噪声协方差矩阵
- $\mathbf{K}_t^{body}$ 为卡尔曼增益

### 3.1.6 生命周期管理

躯干轨迹的生命周期由 `hitCount` 和 `missCount` 共同维护。

当轨迹匹配成功时：

$hitCount \leftarrow \min(hitCount + 1,\; hitThreshold)$，$missCount \leftarrow 0$

当轨迹匹配失败时：

$hitCount \leftarrow \max(hitCount - 1,\; 0)$，$missCount \leftarrow missCount + 1$

当满足 $hitCount \geq hitThreshold$ 时，轨迹进入可输出状态。  
当满足 $missCount \geq missThreshold$ 时，轨迹被删除。

该机制中，`hitCount` 表示轨迹近期命中强度，并被限制在区间 $[0, hitThreshold]$ 内，避免命中计数无限累积；`missCount` 保持连续消失计数的语义，用于在完全确认无匹配时触发删除。

### 3.1.7 驾驶员躯体唯一约束

由于座舱中主驾驶位置应唯一，因此在**躯干输出层**增加驾驶员躯体唯一约束，而不是在所有乘员属性输出完成后再做后处理修正。

若存在多个稳定躯干轨迹被判定为 `DRIVER`，则仅保留一个驾驶员躯体轨迹作为最终输出。可根据躯干历史统计计数选择得分最高者，记为：

$body^{driver} = \arg\max_{b_i \in \mathcal{B}_{driver}} C_i^{driver}$

其中：

- $\mathcal{B}_{driver}$ 表示当前被判定为驾驶员的躯干轨迹集合
- $C_i^{driver}$ 表示第 $i$ 个躯干轨迹的驾驶员累计计数

保留 $body^{driver}$ 作为唯一驾驶员躯体输出，其余候选躯干根据空间位置和累计计数降级为副驾、后排或未知，不再输出为驾驶员。

## 3.2 头部跟踪

头部跟踪在躯干跟踪稳定后启动，作为躯干的附属子轨迹进行初始化；启动后，头部轨迹的存在不再严格依赖躯干轨迹是否持续存在。

### 3.2.1 启动条件

当某一躯干轨迹满足稳定条件 $hitCount \geq T_{hit}^{body}$ 时，则为该躯干创建头部跟踪器，且头部轨迹继承该躯干的 `trackId`。

### 3.2.2 状态建模与预测

头部跟踪采用与躯干相同的状态形式，即  
$\mathbf{x}_t^{head} = [c_x, c_y, w, h, v_x, v_y]^T$。

预测步采用卡尔曼滤波：

$\hat{\mathbf{x}}_{t|t-1}^{head} = \mathbf{F}_{head}\mathbf{x}_{t-1|t-1}^{head}$

$\hat{\mathbf{P}}_{t|t-1}^{head} = \mathbf{F}_{head}\mathbf{P}_{t-1|t-1}^{head}\mathbf{F}_{head}^T + \mathbf{Q}_{head}$

并得到预测框：$Track^{head}_{pred,t} = f(\hat{\mathbf{x}}_{t|t-1}^{head})$。

### 3.2.3 预测与匹配

头部跟踪优先在对应躯干附近区域进行预测与匹配。匹配方式仍采用“预测 + 损失矩阵 + 匈牙利分配”的统一框架。

### 3.2.4 存在解耦机制

头部轨迹启动时依赖躯干轨迹，但一旦初始化完成，其后续存在不再依赖躯干轨迹是否仍存在。

例如，当某乘员的躯干轨迹 `id = 6` 时，其头部轨迹也使用 `id = 6`。若后续躯干因衣物遮挡或局部漏检而消失，只要头部轨迹仍能持续匹配到检测结果，则头部轨迹继续存在，且 `id` 保持为 `6`。

当躯干重新出现并生成新的稳定轨迹，例如新躯干轨迹 `id = 7`，则清除旧的头部和手部轨迹，并基于新的躯干轨迹重新初始化头部和手部跟踪。

### 3.2.5 更新

头部轨迹匹配成功后执行卡尔曼滤波更新步，并维护头部自身的 `hitCount` 与 `missCount`。当头部连续丢失时，仅删除头部子轨迹，不影响其他已存在轨迹。

### 3.2.6 头部唯一原则

在同一乘员区域内，可能同时存在多个头部候选轨迹。为避免异常输出，系统在输出层增加头部唯一原则。

利用躯体框输出的估计头部中心点为 $p^{head} = (x^{head}, y^{head})$。

对于候选头部框 $b_i$，设其中心为 $c_i$，则定义候选距离为：

$d_i^{head} = \|c_i - p^{head}\|_2$

对应得分定义为：

$S_i^{head} = \exp\left(-\frac{d_i^{head}}{\sigma_h}\right)$

仅保留得分最高的头部轨迹作为最终输出，其余候选仅作为内部竞争轨迹保留，不直接输出。

## 3.3 手部跟踪

手部跟踪在躯干稳定后启动，每个躯干轨迹维护两个手部子轨迹：

- `left_hand`
- `right_hand`

由于手部翻转频繁、尺度变化明显，因此手部更新必须同时包含**位置**与**尺度**。同时，由于手部移动快、速度变化大，采用匀加速模型进行建模。

### 3.3.1 启动条件

当躯干轨迹达到稳定条件后，启动左右手跟踪，且左右手轨迹继承该躯干的 `trackId`。

### 3.3.2 左右手先验

左右手依附于所属躯干进行局部跟踪。对于每个躯干，仅在其局部区域内搜索该乘员的手部检测。

左右手身份区分基于：

- 与躯干中心的相对位置
- 与上一帧左手 / 右手轨迹的一致性
- 历史轨迹连续性

### 3.3.3 状态建模

手部目标除位置外，还需要显式维护尺度状态，并引入加速度项。手部状态定义为：

$\mathbf{x}_t^{hand} = [c_x, c_y, w, h, v_x, v_y, a_x, a_y, v_w, v_h]^T$

其中：

- $c_x, c_y$ 表示手部中心位置
- $w, h$ 表示手部框宽高
- $v_x, v_y$ 表示中心速度
- $a_x, a_y$ 表示中心加速度
- $v_w, v_h$ 表示尺度变化速度

观测向量为 $\mathbf{z}_t^{hand} = [c_x, c_y, w, h]^T$。

### 3.3.4 状态预测

左右手分别预测当前时刻状态：

$\hat{\mathbf{x}}_{t|t-1}^{left} = \mathbf{F}_{hand}\mathbf{x}_{t-1|t-1}^{left}$

$\hat{\mathbf{x}}_{t|t-1}^{right} = \mathbf{F}_{hand}\mathbf{x}_{t-1|t-1}^{right}$

其中位置采用匀加速模型预测，尺度采用一阶变化模型预测。解码得到预测框：

$Track^{left}_{pred,t} = f(\hat{\mathbf{x}}_{t|t-1}^{left})$，$Track^{right}_{pred,t} = f(\hat{\mathbf{x}}_{t|t-1}^{right})$

### 3.3.5 数据关联

将预测的左手 / 右手轨迹与当前手部检测构造损失矩阵，并进行匈牙利匹配。

### 3.3.6 存在解耦机制

手部轨迹启动时依赖躯干轨迹，但一旦初始化完成，其后续存在不再依赖躯干轨迹是否仍存在。

例如，当某乘员的躯干轨迹 `id = 6` 时，其左右手轨迹也使用 `id = 6`。若后续躯干消失，只要左右手轨迹仍能持续匹配到检测结果，则其轨迹继续存在，且 `id` 保持为 `6`。

当躯干重新出现并生成新的稳定轨迹，例如新躯干轨迹 `id = 7`，则清除旧的头部和手部轨迹，并基于新的躯干轨迹重新初始化头部和手部跟踪。

### 3.3.7 状态更新

手部跟踪的更新与头部不同，除更新位置外，还需同步更新框尺度。对于匹配成功的手部轨迹，执行卡尔曼滤波更新：

$\mathbf{K}_t^{hand} = \hat{\mathbf{P}}_{t|t-1}^{hand}\mathbf{H}_{hand}^T(\mathbf{H}_{hand}\hat{\mathbf{P}}_{t|t-1}^{hand}\mathbf{H}_{hand}^T + \mathbf{R}_{hand})^{-1}$

$\mathbf{x}_{t|t}^{hand} = \hat{\mathbf{x}}_{t|t-1}^{hand} + \mathbf{K}_t^{hand}(\mathbf{z}_t^{hand} - \mathbf{H}_{hand}\hat{\mathbf{x}}_{t|t-1}^{hand})$

更新后的手部框为 $Track^{hand}_{update,t} = (c_x, c_y, w, h)$。

这样可以更好地适应以下情况：

- 手掌翻转
- 握拳 / 张开
- 前后伸缩导致的投影面积变化
- 部分遮挡下检测框尺度漂移
- 快速抬手、回收手臂等速度突变场景

### 3.3.8 左手唯一原则与右手唯一原则

在同一乘员区域内，可能同时出现多个左手候选轨迹或多个右手候选轨迹。为避免异常输出，系统在输出层增加左手唯一原则与右手唯一原则。

利用人体姿态模型输出的左手腕关键点和右手腕关键点，分别计算候选手部框与对应手腕关键点之间的距离作为得分依据。

设左手腕关键点为 $p^{lw} = (x^{lw}, y^{lw})$，右手腕关键点为 $p^{rw} = (x^{rw}, y^{rw})$。

对于候选手部框 $b_i$，设其中心为 $c_i$，则左手候选距离定义为：

$d_i^{left} = \|c_i - p^{lw}\|_2$

右手候选距离定义为：

$d_i^{right} = \|c_i - p^{rw}\|_2$

对应得分定义为：

$S_i^{left} = \exp\left(-\frac{d_i^{left}}{\sigma_l}\right)$

$S_i^{right} = \exp\left(-\frac{d_i^{right}}{\sigma_r}\right)$

左手候选中仅保留得分最高者作为最终输出，右手候选中仅保留得分最高者作为最终输出，其余候选仅作为内部竞争轨迹保留，不直接输出。

# 4 匈牙利匹配方案

## 4.1 设计目标

匈牙利匹配用于解决以下问题：

- 当前检测应匹配到哪个已有轨迹
- 哪些检测不应匹配任何历史轨迹，而应创建新轨迹
- 哪些历史轨迹当前没有对应检测，应进入丢失状态

系统采用**扩展损失矩阵 + dummy node** 的方式统一建模。

## 4.2 输入集合定义

设当前时刻的检测集合为 $\mathcal{D}_t = \{d_1, d_2, \dots, d_m\}$，历史轨迹预测集合为 $\mathcal{T}_{pred,t} = \{t_1, t_2, \dots, t_n\}$。

其中：

- $m$ 为当前检测数
- $n$ 为历史轨迹数

## 4.3 匹配损失定义

对任意检测 $d_i$ 与轨迹 $t_j$，定义匹配损失为：

$L(i,j) = \lambda_{iou} \cdot L_{iou}(i,j) + \lambda_{dist} \cdot L_{dist}(i,j)$

### 4.3.1 IoU 损失

IoU 损失定义为：

$L_{iou}(i,j) = 1 - IoU(i,j)$

其中，$IoU(i,j)$ 表示检测框与轨迹预测框的交并比。

### 4.3.2 角点距离损失

取框的左上角点距离与右下角点距离的最大值作为框偏移度量：

$d(i,j) = \max(d_{lt}(i,j), d_{rb}(i,j))$

为了使不同尺度目标的距离损失可比较，将该距离使用联合包围框对角线归一化，得到归一化距离损失 $L_{dist}(i,j)$。

### 4.3.3 综合损失

结合现有实现，综合匹配损失为：

$L(i,j) = 10 \cdot (1 - IoU(i,j)) + L_{dist}(i,j)$

其中 IoU 项权重更高，用于优先保证空间重叠一致性；距离项作为几何补偿，用于区分 IoU 接近时的相对位置差异。

## 4.4 dummy loss 机制

为了统一表示“匹配失败优于错误匹配”，在损失矩阵中引入 `dummy node`。

当某个检测与所有历史轨迹的真实匹配损失都较大时，如果 $L(i,j) > L_{dummy}$，则在最优分配中，该检测更倾向于匹配到 `dummy track`，表示**不与任何已有轨迹关联，而是新建轨迹**。

同理，若某个历史轨迹与所有检测的匹配损失都较大，则该轨迹更倾向于匹配到 `dummy detect`，表示**该轨迹当前丢失一次**。

因此，`dummyLoss` 本身就承担了“拒绝劣质匹配”的作用，不再额外引入 gate。只要 `dummyLoss` 设定合理，就能够自然实现以下行为：

- 好匹配优先分给真实轨迹
- 差匹配不强行绑定，转而分给 dummy
- 避免错误匹配污染轨迹状态

## 4.5 扩展损失矩阵构造

为同时处理真实匹配、新建轨迹和轨迹丢失，将原始 $m \times n$ 损失矩阵扩展为方阵 $d \times d$，其中 $d = m + n$。

### 4.5.1 左上区域：真实检测与真实轨迹

$M_{ij} = L(i,j), \quad i \in [0,m-1],\; j \in [0,n-1]$

表示当前检测与历史轨迹之间的真实匹配代价。

### 4.5.2 左下区域：真实检测与伪轨迹

对每个检测 $d_i$，增加一个对应的伪轨迹节点。若检测匹配到伪轨迹，表示该检测不属于任何已有轨迹，需要新建。

该区域采用对角填充 `dummyLoss`，其余位置填充最大值，即：

$M_{i,n+i} = L_{dummy}$

其余非对角位置赋极大值。

### 4.5.3 右上区域：伪检测与真实轨迹

对每个真实轨迹 $t_j$，增加一个对应的伪检测节点。若轨迹匹配到伪检测，表示该轨迹当前未观测到，应增加丢失计数。

该区域采用对角填充 `dummyLoss`，其余位置填充最大值，即：

$M_{m+j,j} = L_{dummy}$

其余非对角位置赋极大值。

### 4.5.4 右下区域：伪检测与伪轨迹

该区域用于补全方阵，全部置为 0，即：

$M_{ij} = 0, \quad i \in [m,d-1],\; j \in [n,d-1]$

## 4.6 匈牙利最优分配

在扩展矩阵 $M$ 上使用匈牙利算法求最小总代价分配。

输出包括：

- `rowAssignments`：每个检测被分配到哪个轨迹 / 伪轨迹
- `colAssignments`：每个轨迹被分配到哪个检测 / 伪检测

这样即可在一次最优化中同时完成：

- 检测与轨迹的一对一匹配
- 未匹配检测的新建决策
- 未匹配轨迹的丢失决策

# 5 短期状态更新

匈牙利匹配完成后，依据匹配结果更新短期状态。

## 5.1 检测匹配到真实轨迹

若某个检测 $d_i$ 的分配结果满足 $rowAssignments[i] < n$，说明该检测匹配到已有轨迹 $t_j$。此时执行以下操作：

1. 根据匹配到的检测生成观测向量
2. 对对应轨迹执行卡尔曼滤波更新步
3. 更新轨迹框与瞬时属性
4. 更新命中计数与丢失计数

匹配成功时，生命周期状态更新为：

$hitCount \leftarrow \min(hitCount + 1,\; hitThreshold)$

$missCount \leftarrow 0$

若跟踪对象为手部，还需要同步更新尺度状态，即更新后的 $w_t$ 和 $h_t$ 直接参与轨迹状态维护。

## 5.2 检测匹配到伪轨迹

检测未匹配到任何已有轨迹，需要新建轨迹：

1. 分配新的 `trackId`
2. 初始化轨迹状态
3. 初始化协方差矩阵
4. 初始化 `hitCount` 与 `missCount`

新建轨迹时可设：

$hitCount = 1$

$missCount = 0$

## 5.3 轨迹匹配到伪检测

轨迹当前未匹配到检测，应执行失配更新：

1. 保留预测状态作为当前轨迹状态
2. 更新命中计数与丢失计数

匹配失败时，生命周期状态更新为：

$hitCount \leftarrow \max(hitCount - 1,\; 0)$

$missCount \leftarrow missCount + 1$

注意，这里并不立即删除轨迹，而是保留一段容忍时间，以应对短时漏检和遮挡。

# 6 长期状态更新

长期状态用于决定轨迹是否进入稳定输出，以及人员属性是否可信。

## 6.1 删除失活轨迹

当轨迹连续丢失达到阈值 $missCount \geq missThreshold$ 时，则将该轨迹从历史轨迹池中删除。

## 6.2 输出稳定轨迹

当轨迹命中计数达到阈值 $hitCount \geq hitThreshold$ 时，则认为该轨迹进入稳定态，可以加入输出结果。

## 6.3 稳定属性更新

对于乘员类型属性，系统不直接采用单帧结果，而是基于多帧累计计数进行投票，降低单帧误判对长期属性的污染。

对于某一轨迹，当 $C_{sum} \geq T_{vote}$ 且 $\frac{C_{max}}{C_{sum}} \geq T_{ratio}$ 时，将该轨迹的稳定属性更新为票数最高的类别。
