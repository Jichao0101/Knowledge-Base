# 背景
## DMS
DMS（DriverMonitoringSystem）驾驶员监测系统通常通过摄像头来获取驾驶员的相关信息。
其主要功能包括：
- 疲劳监测：如果驾驶员出现频繁眨眼、⻓时间闭眼或打哈⽋等疲劳迹象，系统会及时发出警告，提醒驾驶员注意休息，以避免因疲劳驾驶导致的事故。
- 分⼼监测：能够判断驾驶员是否在使⽤⼿机、与乘客交谈过度或者注意⼒不集中在道路上。⽐如，当驾驶员视线⻓时间偏离前⽅道路，系统会发出警⽰。
- ⾝份识别：确认驾驶员的⾝份，以适应不同驾驶员的个性化设置和偏好。
- 情绪监测：分析驾驶员的情绪状态，如愤怒或焦虑，从⽽提供相应的辅助和提醒。
# 要求
DMS所有功能区分⾼灵敏度、标准灵敏度、低灵敏度。在灵敏度切换后的3秒后进⾏算法检测，若当前已触发报警则报警应解除、报警冷却期取消。
## 问题
DMS由多个模型组成,在复杂环境中（遮挡、眯眼、光照、反光眼镜等），各模型输出可能波动或缺失，导致：
- 疲劳检测信号跳变；
- 视线方向不连续；
- 部分特征缺失。
因此，需要 **时序平滑 + 特征融合 + 状态机决策** 的后处理算法。
