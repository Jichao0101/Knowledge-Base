
# 1 需求规范

## 1.1 疲劳特征动作
闭眼动作的判定标准，满⾜以下要求
- 双眼眼睑完全遮盖眼球 
- 从摄像头⻆度观测，眼睛被遮挡的场景为失效场景，不判定为闭眼动作 
打哈⽋动作的判定标准，满⾜以下要求： -
- 张嘴⾼宽⽐（上下嘴唇内缘的最⼩竖直⾼度与嘴⻆⽔平宽度的⽐值）＞0.6 
- 从摄像头⻆度观测，嘴部被遮挡的场景为失效场景，不判定为打哈⽋动作
## 1.2 监测要求 
-  在驾驶员单次闭眼时间达到临界值后，应在1s内进⾏报警
- 从驾驶员打哈⽋动作开始，应在5秒内完成相应的识别,识别的准确率不低于95%
- 能够结合闭眼动作和打哈⽋动作进⾏综合识别分析
- 能够提供识别结果(轻度疲劳、重度疲劳)供其他应⽤调⽤
## 1.3 报警条件
### 1.3.1 触发疲劳报警等级条件
#### 1.3.1.1 轻度疲劳
单次闭眼持续2-3秒（2s≤t＜3s），或 哈⽋检测开启且出现1次3秒以上的哈⽋（t≥3s） 
- 进入轻度疲劳的闭眼时间 FATIGUE_TIME_LOW=2
- 进入重度疲劳的闭眼时间 FATIGUE_TIME_HIGH=3
- 是否开启哈欠检测 FATIGUE_YAWN_DETECT=1
- 哈⽋张嘴持续时间 FATIGUE_YAWN_TIME =3
#### 1.3.1.2 重度疲劳
单次闭眼持续超过3秒（t≥3s） 

> [!NOTE]
> 闭眼时间CNCAP&CIASI要求3秒，IVISTA要求2秒。打哈⽋CIASI2023要求3秒

### 1.3.2 退出疲劳报警等级条件
#### 1.3.2.1 轻度疲劳
1. 5秒内⾮闭眼时⻓超过4秒（t≥4s），或 
2. ⽤⼾DismissAlert#2159，或 
3. 进⼊重度疲劳场景（时序上请先取消轻度疲劳后触发重度疲劳） 
退出疲劳的⾮闭眼时⻓（秒）（监测5秒内数据）FATIGUE_QUIT_TIME=4
#### 1.3.2.2 重度疲劳
1. 5秒内⾮闭眼时⻓超过4秒（t≥4s）FATIGUE_QUIT_TIME=4，或 
2. ⽤⼾DismissAlert#2161 
### 1.3.3 报警冷却期
- 轻度疲劳提醒冷却期：60秒（从上⼀次报警解除开始计算）
   FATIGUE_COOL_TIME_1=60 
- 重度疲劳提醒冷却期：0秒（⽆冷却期） 
  FATIGUE_COOL_TIME_2=0
# 2 算法流程
短窗平滑出干净的帧级结果，长窗累计出秒级结果，更新决策状态
## 2.1 输入输出
### 2.1.1 输入
- 眼部
	- 分类器输出$P_{eye_close}\in[0,1]$
	- 关键点二维坐标
- 嘴部
	- 关键点二维坐标
### 2.1.2 输出
决策状态

## 2.2 特征计算层
### 2.2.1 EAR（Eye Aspect Ratio，眼部纵横比）
用于量化眼睛闭合程度：
$$
EAR = \frac{||p_2 - p_6|| + ||p_3 - p_5||}{2||p_1 - p_4||}
$$
> - 值越小代表眼睛越闭合；
> - 通常清醒状态下 EAR ≈ 0.25~0.35，疲劳或闭眼时 EAR < 0.2。

将EAR转换成闭眼概率
![[Pasted image 20251010131644.png]]
### 2.2.2 MAR（Mouth Aspect Ratio，嘴部纵横比）
用于检测打哈欠行为：
$$
MAR = \frac{||u_2 - l_2|| + ||u_3 - l_3|| + ||u_4 - l_4||}{3||c_L - c_R||}
$$
> - 值越大表示嘴巴张开；
> - 常见阈值：MAR > 0.6 持续 1.5s → 判定打哈欠。

## 2.3 时序平滑层

### 2.3.1 1.异常值抑制
- 对 $x_t$ 应用 Hampel 过滤；
- 若 $|x_t - median| > 3 \times MAD$，则以中值替代。
- 强抑制单帧跳变/关键点抖动

### 2.3.2 2.缺失值保持
- 若连续 $w_i < 0.3$，则冻结上一帧结果；
- 冻结时间上限 $T_{freeze} = 500ms$；
- 超出则标记为“遮挡”状态（OCCLUDED）
### 2.3.3 泄露积分
$$
S_t=\lambda S_{t-1}+(1-\lambda)x_t
$$
$$
\lambda = 1 - e^{-\Delta t/ \tau}
$$
对计算概率做平滑，输出连续概率，消除帧间高频噪声
### 2.3.4 滞回
对平滑后的概率做双阈值滞回，消除低频边界抖动
- **闭眼帧** `eye_flag`：
    
    - 进入：`eye_flag=0` 且 $p ≥ \theta_{on}$ 且 `raw_eye_closed==1`
        
    - 退出：`eye_flag=1` 且 $p ≤ \theta_{off}$ 且 `raw_eye_closed==0`
        
    - $\theta_{on}=0.8$，$\theta_{off}=0.5$
        
- **哈欠帧** `yawn_flag`：
    - 进入：`yawn_flag=0` 且 $p ≥ \theta_{on}$ 且 `raw_mouth_open==1`
        
    - 退出：`yawn_flag=1` 且 $p \leq \theta_{on}$ 且 `raw_mouth_open==0`
        
    - $\theta_{on}=0.8$，$\theta_{off}=0.5$
### 2.3.5 秒级聚合
滞回后得到稳定的帧级标签，相机频率30fps的情况下，维护5s的窗口需要保存150个数据，考虑将标签进行秒级统计，统计过程同样做双阈值滞回，得到秒级标签
如果按整秒输出秒级标签，最坏增加 ≤1 s 延迟，可以维持1s窗口的前提下，将步长设为0.5s，第 k 次更新的滑动窗口覆盖：$[t_k−1.0s,  t_k]$，其中 $t_k=k×0.5s$
相邻窗口重叠0.5s，即每次进出0.5s的数据
$$
N_{eye}^{(k+1)}​=N_{eye}^{(k)}​−N_{eye,out}^{(k)}​+N_{eye,in}^{(k+1)}​
$$


## 2.4 状态机决策
### 2.4.1 状态划分
- NotDrowsy
- LightlyDrowsy
- SevereDrowsy
### 2.4.2 状态转移
switch state
case NotDrowsy
	if 闭眼持续2-3秒（2s≤t＜3s），或 哈⽋检测开启且出现3秒以上的哈⽋（t≥3s）
		if 处于 轻度疲劳提醒冷却期
			break
		else
			state-> LightlyDrowsy
			break
case LightlyDrowsy
	if 5秒内⾮闭眼时⻓超过阈值 或 ⽤⼾DismissAlert
		state->NotDrowsy
		break
	if  闭眼持续超过3秒
		if 处于 重度疲劳提醒冷却期（无冷却期）
			break
		else
			state->SevereDrowsy
			break
case SevereDrowsy
	if  5秒内⾮闭眼时⻓超过阈值 或 ⽤⼾DismissAlert
		state->LightlyDrowsy
		break
	else
		break
