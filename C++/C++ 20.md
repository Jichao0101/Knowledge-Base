# 1 概念
## 1.1 原子智能指针
std::atomic<shared_ptr>保证指针本身的读写是原子的，即多线程之间不会产生指针层面的数据竞争
```
int main() {

    std::atomic<std::shared_ptr<int>> atomicPtr;

    std::shared_ptr<int> ptr = std::make_shared<int>(42);

    atomicPtr.store(ptr); //原子写入

    std::shared_ptr<int> loadedPtr = atomicPtr.load(); //原子读取

    std::cout << "Value: " << *loadedPtr << std::endl;

}
```
但它不能保证被管理的对象本身是线程安全的。对象是否线程安全取决于资源自身的管理方式，即使用原子智能指针管理同一个对象，如果多个线程同时调用这些资源的非const成员函数，而这些函数内部修改共享状态，且没有加锁或同步机制，那么资源本身依然不是线程安全的
## 1.2 移动语义
### 1.2.1 零原则
编写类时，不需要定义非虚析构函数（例外，基类应该定义公共虚析构函数，或受保护的非虚析构函数），也不需要定义复制构造函数、move构造函数、复制赋值操作符、move
赋值操作符。应该使用智能指针、标准库类和容器来管理资源。
### 1.2.2 返回局部对象
- C++17及以后：强制启用RVO(return value optimization)，编译器直接在调用者的栈上构造对象，没有拷贝也没有移动
- C++11/14：如果RVO没触发，编译器会自动尝试调用移动构造函数（如果可用），否则退回拷贝
## 1.3 编译时计算
### 1.3.1 constexpr
### 1.3.2 constexpr函数
constexpr 修饰函数，表示这个函数**可以**在编译期执行（compile-time evaluated）。但是：它**不是必须在编译期执行**，而是 **“有资格在编译期执行”**
- 当 `constexpr` 函数的**实参全是编译期常量**时，编译器会：
	- **直接执行函数逻辑**
	- 把结果**常量折叠（constant folding)** 进目标代码
	- 生成的汇编中，结果通常是 `mov` 指令赋常数。
- 如果参数不是编译期常量：
	- 编译器会保留函数定义；
	- 在运行时正常调用（就像普通函数一样）
也就是说，constexpr 函数兼具编译期函数与普通函数的两种能力
### 1.3.3 constexpr变量
constexpr表示**编译期常量**，必须**在编译期能确定值**，编译器在编译阶段直接计算；只有 `constexpr` 才能被 `if constexpr`、模板参数、编译期推导机制所使用
```
constexpr int a = 5;            // ✅ 编译期常量 
constexpr int b = square(3);    // ✅ 可在编译期计算 
constexpr int c = getValue();   // ❌ 编译期不可确定，报错
```
const只是**只读变量**，表示**运行时不可修改**，但初始化时**可以用运行时值**，存储在静态或栈区。
```
const int a = getValue();  // ✅ 允许运行时初始化
```
## 1.4 Type-rich 编程
### 1.4.1 type_traits
type_traits 是 C++ 标准库中的一个头文件，主要用于在编译期对类型进行分析、判断、转换和操作
#### 1.4.1.1 is_integral
```
#include <iostream>

  

template <typename T>

struct my_is_integral {

    static const bool value{false};

};

  

template <>

struct my_is_integral<int> {

    static const bool value{true};

};

  

template <>

struct my_is_integral<char> {

    static const bool value{true};

};

  

template <typename T>

constexpr bool my_is_integral_v = my_is_integral<T>::value;

  

int main() {

    std::cout << std::boolalpha;

    std::cout << "my_is_integral_v<int>: " <<  my_is_integral_v<int> << std::endl;     // true

    std::cout << "my_is_integral_v<char>: " <<  my_is_integral_v<char> << std::endl;   // true

}
```

#### 1.4.1.2 true_type
```
#include <iostream>

#include <type_traits>

  

template <int M, int K, int S>

struct MksUnit {

    enum { meter = M, kilogram = K, second = S };

};

  

//泛型模板默认“不是MksUnit类型”

template <typename T>

struct IsMksUnitType : std::false_type {};

  

//偏特化版本：MksUnit类型

template <int M, int K, int S>

struct IsMksUnitType<MksUnit<M, K, S>> : std::true_type {};

  

//语义上等价于IsMksUnitType<T>::value == true

template <typename T>

concept MksUnitType = IsMksUnitType<T>::value;

  

// 它的模板参数 T 需要满足 MksUnitType 约束（即必须是一个单位类型）

// 这个类本质上只是封装了一个带单位的数值（magnitude_）

template <typename T> requires MksUnitType<T>

class Value {

public:

    constexpr explicit Value (const long double magnitude) noexcept : magnitude_(magnitude) {}

    constexpr long double getMagnitude() const noexcept { return magnitude_; }

private:

    long double magnitude_{0.0};

};

  

int main() {

    using Acceleration = Value<MksUnit<1, 0, -2>>; // m/s^2

    constexpr Acceleration a{9.81};

    std::cout << "a = " << a.getMagnitude() << " m/s^2";

}
```
### 1.4.2 concept
concept是 C++20 引入的语法糖，用来在**编译期对模板参数做布尔逻辑约束**。它本质上就是一个返回 `bool` 的编译期表达式（constexpr context）
假设定义了concept
```
//某类型 Class 必须有一个常量成员函数 getTypeOfInstanceAsString()，且它的返回类型必须是 std::string

template <typename Class>

concept ClassWithConstCallableMethod = requires(Class& a) {

    {a.getTypeOfInstanceAsString()} -> std::same_as<std::string>;

};
```
1. 在模板参数列表中直接使用
```
template <ClassWithConstCallableMethod T>

void printTypeOfInstance(const T& instance) {

    std::cout << instance.getTypeOfInstanceAsString() << '\n';

}
```

2. 使用`requires`语句
```
template <typename T> requires ClassWithConstCallableMethod<T>

void printTypeOfInstance2(const T& instance) {

    std::cout << instance.getTypeOfInstanceAsString() << '\n';

}
```
3. 在函数体中使用`if constexpr`
```
template <typename T>

void checkTypeOfInstance(const T& instance) {

    if constexpr (ClassWithConstCallableMethod<T>) {

        std::cout << instance.getTypeOfInstanceAsString() << '\n';

    } else {

        std::cout << "The type does not have the required method.\n";

    }

}
```
## 1.5 函数式编程
### 1.5.1 std::bind
std::bind是函数及其参数的一个绑定包装器，可以将实际值绑定到函数、函数指针、仿函数的一个或所有参数上。它允许在编程中使用局部应用，即只有一部分函数参数绑定到值或变量，另一部分不绑定，未绑定的参数由占位符`_1`,`_2`替换
```
#include <functional>

#include <iostream>

  

[[nodiscard]] constexpr double multiply(const double multiplicand, const double multiplier) noexcept {

    return multiplicand * multiplier;

}

  

int main() {

    const auto multiplyWith10 = std::bind(multiply, 10.0, std::placeholders::_1);

    std::cout << "10.0 * 3.5 = " << multiplyWith10(3.5) << '\n';

}
```
### 1.5.2 Lambda模板
使用lambda模板，防止错误实例化或使用lambda表达式
```
#include <concepts>

#include <iostream>

#include <string>

  

template <typename T>

concept Number = std::integral<T> || std::floating_point<T>;

  

int main() {

    auto add = [] <Number T>(T a, T b) { return a + b; };

    const std::string string1 {"Hello, "};

    const std::string string2 {"World!"};

    auto result1 = add(1, 2); // OK: T 被推导为 int

    auto result2 = add(1.5, 2.5); // OK: T 被推导为 double

    // auto result3 = add(string1, string2); // 错误: std::string 既不是整数类型也不是浮点类型

}
```
### 1.5.3 高级函数
高级函数将一个或多个其他函数作为参数，可以返回一个函数
1. Map
	Map将一个运算符函数应用于列表的每个元素，标准库算法`std::transform`
2. Filter
	从列表中删除不满足谓词条件的元素，标准库算法`std::remove_if`
3. Reduce
	在值列表上应用二元运算符获得单一结果，标准库算法`std::accumulate`
