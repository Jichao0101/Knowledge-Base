
模板实例化在编译期（预处理之后，实际编译之前），只能解决编译期确定的问题，倾向于用构建效率换运行效率（前置优化）

# 1 模板特例化

为某些特定类型提供一个与通用模板不同的实现。当模板参数匹配特例化类型时，编译器会优先使用特例化版本，否则使用通用版本

```
//通用模板

template<typename T>

struct ArgType{

using type = const T&;

};


//完整特例化

template<>

struct ArgType<int>{

using type = int; 

};

//偏特化 partial specialization

template<typename T>

struct ArgType<T*>{

using type = T*;

};

```

# 2 SFINAE

substituion faiure is not an error 编译器在模板实例化时，如果遇到匹配失败（无法按此方法实例化），不会马上报错，而是继续匹配其他模板实现

```
template<bool Cond, typename T = void>

struct enable_if{};


template<typename T>

struct enable_if<true, T>{

using type = T;

};


template<typename T>

struct IsLessThanPtr{

constexpr static bool val = sizeof(T) <= sizeof(void *);

};



//模板全局常量

template<typename T>

constexpr bool IsLessThanPtr_v = IsLessThanPtr<T>::val;


template<typename T>

void f(typename enable_if<IsLessThanPtr_v<T>, T>::type t){

std::cout << 1 << std::endl;

}


template<typename T>

void f(typename enable_if<!IsLessThanPtr_v<T>, T>::type& t){

std::cout << 2 << std::endl;

}
```
