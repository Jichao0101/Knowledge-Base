# 1 虚函数
在c++中，虚函数用于实现多态。在基类中将函数声明为虚函数后，通过基类指针或引用调用该函数时，程序会检查实际指向的对象类型。如果指向的是派生类对象，并且该派生类重写了虚函数，则调用重写后的版本；否则，调用基类中的虚函数版本。
当基类的虚函数不是纯虚函数，派生类可以选择是否重写该函数。如果派生类没有提供重写，基类的虚函数会作为默认实例。
当基类的虚函数是纯虚函数，所有派生类必须实现该函数，否则派生类也将成为抽象类，无法直接实例化。

# 2 奇异递归模板模式
然而虚函数带有一定的开销，因为每次调用时需要通过虚函数表查找真正要调用的函数。
## 2.1 概念
CRTP(Curiously Recurring Template Pattern)是一种模板编程模式，使用模板参数让基类在编译时调用派生类的实现。这种模式可以让基类定义一些接口函数，这些接口函数在派生类中实现，从而达成类似虚函数的效果，但不需要虚函数的动态绑定和开销。但CRTP是静态多态，编译器在编译时就确定了调用的函数，因此无法在运行时动态调用哪个派生类的方法。
## 2.2 示例
使用CRTP抽象出Vector计算的共同方法，减少代码冗余
```
#include <iostream>

template<typename T>
struct VectorBase
{
	T& underlying() {return static_cast<T&>(*this);}
	const T& underlying() const {return static_cast<const T&>(*this);}

	inline T& operator+=(const T& rhs)
	{
		this->underlying() = this->underlying() + rhs;
		return this->underlying();
	}

	inline T& operator-=(const T& rhs)
	{
		this->underlying() = this->underlying() - rhs;
		return this->underlying();
	}
};

struct Vector3 : public VectorBase<Vector3>
{
	float x, y, z;

	Vector3() = default;
	Vector3(float _x, float _y, float _z)
	{
		x = _x;
		y = _y;
		z = _z;
	}
};

inline Vector3 operator+(const Vector3& lhs, const Vector3& rhs)
{
	Vector3 result;
	result.x = lhs.x + rhs.x;
	result.y = lhs.y + rhs.y;
	result.z = lhs.z + rhs.z;
	return result;
}

inline Vector3 operator-(const Vector3& lhs, const Vector3& rhs)
{
	Vector3 result;
	result.x = lhs.x - rhs.x;
	result.y = lhs.y - rhs.y;
	result.z = lhs.z - rhs.z;
	return result;
}

int main()
{
	Vector3 v0(6.0f, 5.0f, 4.0f);
	Vector3 v1(5.0f, 4.0f, 3.0f);

	v0 -= v1;
	std::cout << v0.x << v0.y << v0.z <<std::endl;
}
```
