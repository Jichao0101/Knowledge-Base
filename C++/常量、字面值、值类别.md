# 1 常量、字面量
字面量是写在代码里的固定值本身，没有变量名。字面量在编译期就被固定了，直接嵌在代码段或常量池里，没有单独的内存
常量是变量名+值不能改的组合，可以是字面量，也可以是其他表达式的结果，关键在于其不可变。常量可能是const修饰的变量，constexpr编译期常量。常量有独立内存，但有可能被编译器优化成立即数，也可能分配在.rodata只读段
- 自定义字面量
```
#include <cstdint>
constexpr uint64_t operator ""_killo(unsigned long long n){
    return n * 1024;
}
int main(){
    uint64_t capacity = 5_killo;
}

```
- 字符串字面量
```

```

# 2 值类别
## 2.1 左值、右值、将亡值

C++ 中的值类别包括 lvalue（左值）、xvalue（将亡值）、prvalue（纯右值）。它们进一步构成了 glvalue（泛左值）和 rvalue（右值）这两个总类：
- glvalue = lvalue + xvalue：有“地址”的表达式
- rvalue = prvalue + xvalue：将被销毁/移动的表达式
在 C++ 中，左值（lvalue）与右值（rvalue）的核心区别在于：左值表示一个具名且可寻址的内存位置，而右值表示一个临时值或表达式结果，通常没有独立存储。并不是“可修改”才是左值。许多不可修改的对象（如 const int、字符串字面量）仍然是左值；而右值即使类型非 const，也通常不具备可寻址性，无法赋值。将亡值（xvalue）通常来源于 std::move 或函数返回右值引用，表示资源即将被转移的对象，具有可寻址性，是可修改的 glvalue
修改一个对象的本质在于编译器是否允许你在语义上进行写操作，同时底层内存空间也必须是可写的。例如栈、堆、全局变量常驻内存空间是可读写的；而字符串字面量、常量通常位于只读段，不可安全修改。虽然从底层技术上可以通过 const_cast、指针运算等手段绕过语言的限制修改对象，但这些行为属于未定义行为（UB），不能被依赖
## 2.2 移动语义、右值引用
```
void f(String s);//会触发拷贝，浪费空间
void f(const String& s);//无法修改变量
void f(String& s);//无法传递右值（包括字面量），例如f("ABC")

移动语义，创建右值引用
void f(String&& s);
String s = "ABC";
f(std::move(s)); //相当于f(static_cast<String&&>(s)),s为将亡值

//移动构造函数，用将亡值创建新对象，复用原本的数据->浅复制
String::String(String&& s) : buffer(s.buffer), size(s.size), capacity(s.capacity) {
    s.buffer = nullptr;
    s.size = 0;
    s.capacity = 0;
}
```

右值引用（T&&）可以绑定到将亡值或纯右值，它并不会默认产生拷贝。若绑定到 prvalue，其生命周期将被延长，可以安全地修改引用的对象。若绑定到将亡值，即用函数参数T&&来接收xvalue，即可把将亡值的资源转移到新的对象中
```
String test(){
    return String("ABC");
}
String s1 = "ABC";
String&& s2 = std::move(s1);右值引用绑定将亡值
String&& s3 = test();右值引用绑定纯右值

```
在表达式中，具名的右值引用是左值（因为具名对象始终是左值），不具名的右值引用表达式（如 std::move(x)）是将亡值（xvalue）
