
# 1 成员函数的重载
c++中的成员函数有三种：普通成员函数，virtual虚函数，const成员函数
成员函数被重载的特征：
- 具有相同的作用域（即同一个类定义中）
- 函数名字相同
- 参数类型，顺序或数目不同（包括const参数和非const参数）
在类中的函数，都会自动添加一个自身类的指针this，因此const函数可以和普通函数重载
```
void func(int a) === void func(Base* this, int a)
virtual void func(int a) === virtual void func(Base* this, int a)
void func(int a) const === void func(const Base* this, int a) const
```
# 2 成员函数的继承
继承是指派生类获得了基类的所有非私有成员函数和成员变量，包括普通函数和虚函数。
继承过程中，派生类的虚函数表vtable会包含从基类继承来的虚函数，未被覆盖的虚函数指向基类的实现。
vtable（虚函数表）：编译器为每个具有虚函数的类生成一个虚表（函数指针数组）表中列出该类所有虚函数的最终实现地址。每个类（Base, Derived）各有一张独立的虚表
vptr（虚表指针）：每个有虚函数的对象实例里，都会有一个 隐藏成员指针（vptr），指向对应类的 vtablevptr 的位置。通常在对象内存布局的最前面
```
class Base {
public:    
    virtual void hello();   
    virtual ~Base();
};
class Derived : public Base 
{public:    
    void hello() override;
    ~Derived();};
//Base 有一张 Base_vtable，包含：Base::hello,Base::~Base
//Derived 有一张 Derived_vtable，包含：Derived::helloDerived::~Derived
//构造时先构造 Base，所以 vptr 先指向 Base 的虚表；当构造 Derived 完成时，vptr 再被更新为指向 Derived 的虚表
/*
构造阶段：
1. 执行 Base 构造函数
   - vptr → Base_vtable
2. 执行 Derived 构造函数
   - vptr → Derived_vtable（覆盖原来的）

完成后对象布局：
[ vptr → Derived_vtable ]
[ Base 成员数据 ]
[ Derived 成员数据 ]
*/

Base* b = new Derived;
delete b;  
//如果 Base::~Base 不是 virtual：编译器直接调用 Base::~Base（静态类型决策，不查 vptr）,不调用 Derived::~Derived,导致子类资源泄漏
//如果 Base::~Base 是 virtual：delete 操作会读取 b 中的 vptr，并跳转到虚表中的 ~Derived 实现,~Derived 内部会再自动调用 ~Base
```

## 2.1 private继承、protected继承
私有继承（private inheritance）：主要用于复用实现，而非表现出"是一个"（is-a）关系。基类的 public 和 protected 成员都降为 private。不支持外部的基类-派生类转换（即不能通过基类指针或引用访问子类对象）
```
class Alloc {
public:
    void f();
};

struct Container1 : private Alloc {  // 私有继承
    uint64_t ch;
    void f() { Alloc::f(); } // 访问父类方法，需要显式指明 Alloc::f()
};

struct Container2 {  // 组合
    uint64_t ch;
    Alloc al;
    void f() { al.f(); } // 访问成员对象的方法，需要通过对象al来调用
};
```
受保护继承（protected inheritance）：允许子类继续使用基类功能，但对外隐藏。基类的 public 和 protected 成员都降为 protected。

# 3 成员函数的覆盖
覆盖指派生类重新实现了基类的成员函数，特征是
- 不同的作用域（分别位于基类和派生类中）
- 函数名称相同
- 参数列表完全相同
- 基类函数必须是虚函数
在派生类的虚函数表中，覆盖的函数会替换基类的虚函数。
# 4 成员函数的隐藏
- 隐藏是指派生类的成员函数遮蔽了与其同名的基类成员函数，规则如下：
	- 派生类的函数与基类的函数同名，但是参数列表有所差异。此时不论有无virtual关键字，基类的函数在派生类中将被隐藏。（区分重载）
	- 派生类的函数与基类的函数同名，参数列表也相同，但是基类函数没有virtual关键字，此时基类的函数在派生类中将被隐藏。（区分覆盖）

- 编译器根据指针或引用的类型来决定调用哪个函数：
	- 非虚函数的情况下，函数的绑定（即确定调用哪一个版本的函数）是在编译时完成的（静态绑定），而非在运行时。基类指针指向派生类对象时，编译器只知道这是一个基类指针，因此会选择调用基类版本的同名函数，而不会检查派生类中是否有同名的隐藏版本
	- 如果该函数是虚函数，则会在运行时根据指向的对象的实际类型来决定调用派生类的函数版本，从而实现多态（动态绑定）
```
#include <iostream>

class Animal
{
public:
	void Speak() {std::cout << "Animal speaks" << std::endl;}
};

class Dog : public Animal
{
public:
	void Speak() {std::cout << "Dog speaks" << std::endl;}
};

int main()
{
	Dog dog;
	dog.Speak();
	Animal* animalPtr = &dog;
	animalPtr->Speak();
	std::cin.get();
}
//输出结果为：Dog speaks   Animal speaks
```
