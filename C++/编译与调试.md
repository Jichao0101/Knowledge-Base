- disable breakpoints

# 3 CMAKE
## 3.1 语法
- 基本语法格式：指令（参数1 参数2...），参数之间使用空格或分号分开
- 指令是大小写无关的，参数和变量是大小写相关的
- 变量使用${}方式取值，但是在IF语句中直接使用变量名
## 3.2 常用指令
```
cmake_minimum_required  指定CMake的最小版本要求
cmake_minimum_required(VERSION 3.0)


#project 定义工程名称，并指定支持的语言  project(Projectname[CXX][C][Java])
project(TEST)

#set 显式定义变量 set(VAR[VALUE])
set(SRC main.cpp hello.cpp)

 include_directories(dir1 dir2 ...) 向工程添加多个头文件搜索路径，相当于指定g++编译器的-I选项
include_directories(/usr/include/myincludefoler  ./include)

# link_directories(dir1 dir2 ...) 向工程添加多个库文件搜索路径，相当于指定g++编译器的-L选项
link_directories(/usr/lib/mylibfolder ./lib)

# add_library(libname [SHARED|STATIC|MODULE] source1 source2 ...) 生成库文件
add_library(mylib SHARED ${SRC})

# add_compile_options(option1 option2 ...) 向工程添加多个编译选项，相当于指定g++编译器的-D选项
add_compile_options(-DDEBUG -std=c++11 -Wall)

# add_executable(exe_name source1 source2 ...) 生成可执行文件
add_executable(myexe ${SRC})

# target_link_libraries(exe_name libname1 libname2 ...) 为可执行文件链接多个库文件，相当于指定g++编译器的-l选项
target_link_libraries(myexe mylib)

# add_subdirectory(source_dir[binary_dir]) 向当前工程添加存放源文件的子目录
add_subdirectory(subdir)

# aux_source_directory(dir variable) 将指定目录下的所有源文件添加到变量中
aux_source_directory(subdir SRC)
add_executable(myexe2 ${SRC})
```

## 3.3 CMake常用变量
```
●CMAKE_CXX_FLAGS g++编译选项
# 在编译选项后追加-std=c++11
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -std=c++11")

●# CMAKE_BUILD_TYPE 编译类型 Debug Release
set(CMAKE_BUILD_TYPE Debug)

● CMAKE_BINARY_DIR，PROJECT_BINARY_DIR， <PROJECT_NAME>_BINARY_DIR ，三个变量指代的内容一致，如果是in source build，指的就是工程顶层目录；如果是out of source build，指的就是工程编译发生的build目录。
●CMAKE_SOURCE_DIR, PROJECT_SOURCE_DIR, <PROJECT_NAME>_SOURCE_DIR 三者无论使用何种编译方式，都是工程顶层目录
## 编译方式
```
### 3.3.1 两种编译规则
- 包含源文件的子文件夹包含CMakeLists.txt文件，主目录的CMakeLists.txt通过add_subdirectory添加子目录即可
- 包含源文件的子文件夹不包含CMakeLists.txt文件，子目录的编译规则体现在主目录的CMakeLists.txt文件
### 3.3.2 两种构建方式
- 内部构建(in source build），不推荐使用。内部构建会在同级目录下产生一大堆中间文件，和工程源文件放在一起显得杂乱无章
- 外部构建(out-of-source build)，将编译输出文件与源文件放到不同目录中
```
# 当前目录下创建build文件夹
mkdir build 
cd build
# 编译上级目录的CMakeLists.txt，生成Makefile和其他文件
cmake ..
# 执行make命令生成target
make
```
