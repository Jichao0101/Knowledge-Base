# 2 内存泄漏
程序在运行过程中分配了内存，但没有正确释放，导致内存资源无法被重新利用
- 动态内存未手动释放
# 3 程序不确定性行为
- 访问不存在的字典键：使用operator[]访问map[key]，如果键不存在，std::map或std::unordered_map会自动插入一个默认值到字典中，返回该默认值
	- 解决方案：使用find()方法检查键是否存在
- 未初始化的指针或成员变量内用来控制循环、判断或内存分配
