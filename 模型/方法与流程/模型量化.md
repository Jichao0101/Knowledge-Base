# 1 PTQ（训练后量化）

## 1.1 目的与适用范围

PTQ（训练后量化）的目标是：  

- 不重新训练模型
    
- 用低比特逼近浮点模型数值行为

PTQ 适用于：
- 已完成训练、权重冻结的模型
- 对推理性能/资源有约束的部署场景
- 不具备或不希望引入量化感知训练（QAT）的情况

## 1.3 PTQ 的核心任务

PTQ 并不通过学习来优化 accuracy，其核心任务是：

1. **刻画将被量化张量的数值分布（activation / weight）**
    
2. **为每个量化点建立稳定的量化坐标系**
    
3. **在固定坐标系下最小化数值近似误差**
    

> PTQ 的输出不是“更好的模型”，而是“一套量化规则”。

---

## 1.4 Calibration

### 1.4.1 Calibration 的目的

Calibration 的目标不是评估模型性能，而是：

- 测量各量化点张量的数值分布
- 为量化建立合理的数值映射关系
- 减少浮点 → 低比特的近似误差

Calibration 不需要标签（label-free），只决定量化参数，不改变模型参数

---

### 1.4.2 校准对象

Calibration 的统计对象是“将来会被量化的张量”，而不是算子本身

#### 1.4.2.1 Weight（静态）

- 权重在训练后固定
- 可直接统计：
  - min / max
  - 直方图
- 常用对称量化
- 通常支持 per-channel 量化（尤其卷积）

#### 1.4.2.2 Activation（动态）

- 依赖输入数据
- 需通过 calibration set 跑一遍模型收集统计量
- 常见统计：
  - min / max
  - percentile
  - histogram（用于 KL / MSE）

---

### 1.4.3 Calibration Set

Calibration set 的要求：
- 来自训练集或与训练集同分布
- 覆盖典型输入场景
- 数量不必很大（几十到几百即可）

Calibration set 的职责：
- 估计数值分布
- 确定量化参数

**Calibration set 不用于评估泛化性能。**

---

## 1.5 量化参数的确定

### 1.5.1 线性量化模型

典型线性量化公式：

- Quantize：

$q = \mathrm{clip}(\mathrm{round}(x/s) + z)$


- Dequantize：

$\hat{x} = s \cdot (q - z)$


其中：
- `s`：scale（量化步长）
- `z`：zero-point（零点）
- clip：截断到整数表示范围

---

### 1.5.2 对称 vs 非对称量化

- **对称量化**
  - zero-point = 0
  - 常用于 weight
  - 硬件友好

- **非对称量化**
  - zero-point ≠ 0
  - 更好覆盖偏移分布
  - 常用于 activation

---

### 1.5.3 Clipping 策略

直接使用 min/max 易受 outlier 影响，常见替代策略：

- Percentile（如 99.9%）
- KL-divergence
- MSE 最小化

目标是在：
- 舍入误差（scale 过大）
- 截断误差（clip 过多）

之间取得平衡。

---

### 1.5.4 Per-Tensor vs Per-Channel

- **Per-tensor**
  - 实现简单
  - 精度相对较差

- **Per-channel**
  - 每个通道独立 scale
  - 对卷积权重显著提升精度
  - 依赖硬件/引擎支持

---

## 1.6 PTQ 后的评估

### 1.6.1 Calibration Set（数值近似验证）

关注：
- 量化前后激活/权重误差
- 中间层数值偏差
- 是否出现异常饱和或截断

不关注最终 accuracy。

---

### 1.6.2 Test Set（泛化验证）

Test set 用于回答：

> **量化后的模型在未见数据上是否保持决策一致性？**

关注指标包括：
- Accuracy / Precision / Recall
- 与 FP32 模型的输出一致性
- 类别偏移或系统性误差

Calibration set 无法替代 Test set。

---

## 1.7 常见问题与风险点

- 输入前处理不等效（最常见）
- Calibration 数据分布不代表线上
- Resize / 颜色空间实现不一致
- scale 被 outlier 拉大
- 激活量化策略过于粗糙
- 输出层或敏感层量化误差放大

---

## 1.8 总结

PTQ 的本质是：

> **在保证输入等效的前提下，  
> 用有限比特尽可能逼近浮点模型的数值行为。**

# 2 QAT

## 2.1 QAT的目的与定位

目标是：
- 在训练阶段显式引入量化误差
- 通过优化模型参数，使其对量化噪声具有鲁棒性
- 弥补PTQ在复杂模型或敏感结构上的精度损失

QAT不是重新定义量化规则，而是：
	在既定量化规则下，重新寻找更鲁棒的参数

## 2.2 QAT 与 PTQ 的根本区别

| 维度       | PTQ         | QAT             |
| -------- | ----------- | --------------- |
| 是否训练     | ❌           | ✅               |
| 量化误差来源   | 推理阶段        | 训练阶段显式引入        |
| scale 来源 | calibration | calibration（初值） |
| 优化对象     | 无           | weight          |
| 适用模型     | 量化敏感度低      | 量化敏感度高          |

## 2.3 伪量化模型(fake quant model)

### 2.3.1 定义

在FP数据类型下，显式模拟INT8数值约束的训练模型

`FP → Quantize → DeQuantize → FP`

- dtype 始终为 FP
    
- 数值被周期性拉回量化网格

### 2.3.2 伪量化的作用

伪量化模型的作用包括：

1. **在 forward 中真实引入量化误差**
    
2. **使 loss 反映量化后的数值行为**
    
3. **为反向传播保留可训练性**

## 2.4 STE（Straight-Through Estimator）

### 2.4.1 为什么需要 STE

真实量化操作（round / clip）不可微。  
若直接求导，梯度为 0，训练无法进行。

STE 的做法是：

- forward：使用真实量化函数
    
- backward：人为定义近似梯度（通常为 1）
    

### 2.4.2 STE 的实际作用

STE 并不是“忽略量化误差”，而是：

> **忽略量化函数的不可微性，但不忽略量化误差对 loss 的影响。**

因此：

- 量化误差通过 forward 进入 loss
    
- 模型参数通过梯度更新，逐步适应这种误差

## 2.5 QAT 的典型流程

```
FP32 预训练模型
 → Calibration（确定量化坐标系）
 → 构建伪量化模型（插入 Q/DQ）
 → QAT 微调（更新 weight）
 → 导出量化模型 / 部署

```

> Calibration 在 QAT 中的角色是：  
> **定义量化环境，而非被训练对象。**

## 2.6 QAT 的适用与失效场景

### 2.6.1 适合 QAT 的情况

- 激活分布尖锐或不稳定
    
- 深层网络累积量化误差
    
- PTQ 精度明显下降
    
- backbone / neck 中的线性算子
    

---

### 2.6.2 QAT 无法解决的问题

- accumulator 位宽不足
    
- 硬件饱和 / 溢出
    
- 非线性放大误差（softmax / exp / div）
    
- 算子不支持或编译 fallback
    

---

## 2.7 QAT 的工程注意事项

- QAT 需基于 **最终部署图（融合后算子图）**
    
- Q/DQ 插入位置必须与后端编译一致
    
- scale 通常固定或半固定
    
- 训练不宜过长，避免过拟合量化噪声
    

---

## 2.8 QAT 总结

QAT 的本质是：

> **在固定量化坐标系下，  
> 通过参数优化，使模型对量化噪声更加鲁棒**
