# 1 整体工程视角

一个可量产的深度学习项目，必须明确分层：

```
数据层 → 前处理 → 模型 → 损失函数 → 优化器 → 训练策略
                                     ↓
                              pth（权重）
                                     ↓
                              ONNX（计算图）
                                     ↓
                              HBM / TRT
                                     ↓
                         SDK 前后处理 + 推理

```

**关键原则：**

- `.pth` 不是模型，只是参数
    
- **模型 = 结构代码 + forward 逻辑**
    
- ONNX / HBM 需要**静态算子图**
    

___

# 2 数据与预处理（Preprocessing）

##  预处理的工程定位

> **预处理不是“辅助步骤”，而是模型的一部分（但不在模型图里）**

训练 / 导出 / SDK **必须一致**。

---

##  常见视觉预处理流程

### 输入数据来源

- Camera / Video / Image
    
- 格式：BGR / RGB / NV12 / YUV420
    

---

###  颜色空间转换

`NV12 / YUV → RGB（或 BGR）`

⚠️ 错误的颜色空间 = 模型性能直接下降  
（这是工程中最常见的“模型没效果”原因之一）

---

###  Resize / Crop / Letterbox

- 分类模型：  
    `Resize → CenterCrop / ROI Crop`
    
- 检测模型：  
    `Resize + Padding (letterbox)`
    

⚠️ **训练与推理必须使用同一种几何变换**

---

###  Normalize

典型 ImageNet 归一化：

```
mean = (0.485, 0.456, 0.406) 
std  = (0.229, 0.224, 0.225)
```

工程原则：

- Normalize **永远不进模型**
    
- 但 **必须被写进“模型规范文档”**
    

---

###  Tensor 格式

`HWC → CHW uint8 → float32 / float16`

---

##  预处理一致性原则

> **如果训练前处理 ≠ SDK 前处理，  
> 模型表现没有任何讨论价值**

---

# 3 模型结构与计算图（Model & Graph）

## 3.1 模型结构的工程定义

`Backbone + Neck（可选） + Head`

- Backbone：特征提取（ResNet / MobileNet / CSP）
    
- Head：
    
    - 分类：Linear / Conv + Pool
        
    - 检测：BBox + Class + Objectness
        

---

## 3.2 计算图与算子拓扑

- 模型在 PyTorch 中是 **动态图**
    
- 导出 ONNX 时，**一次 forward 会被冻结成静态图**
    

**工程约束：**

- ❌ Python if / for 控制结构（依赖数据）
    
- ❌ shape 依赖 runtime tensor
    
- ✅ 所有逻辑必须转为 tensor 运算
    

---

## 3.3 dummy input 的工程意义

```
dummy = torch.randn(1, 3, H, W) 
torch.onnx.export(model, dummy, ...)

```
dummy input 用来：

- 固定输入 shape
    
- 决定 forward 路径
    
- 生成完整算子图
    

📌 **模型结构不应依赖输入内容，只应依赖输入 shape**

# 4 损失函数（Loss Function）

## 4.1 工程视角下的损失函数本质

> **损失函数不是“算错了多少”，  
> 而是“如何惩罚错误，尤其是自信的错误”。**

---

## 4.2 分类任务：交叉熵（Cross Entropy）

### 4.2.1 多分类交叉熵

$$\mathcal{L} = -\log p_{\text{true}}$$​

工程含义：

- 预测正确且自信 → loss → 0
    
- 预测错误且自信 → loss → ∞（理论上）
    

---

### 4.2.2 梯度的关键性质

softmax + CE：

$$\frac{\partial \mathcal{L}}{\partial z_k} = p_k - y_k$$​

工程意义：

- 错得越自信，梯度越大
    
- 训练稳定、收敛快
    
- 非常适合分类任务
    

---

### 4.2.3 Label Smoothing

目的：

- 削弱“无限确信”
    
- 提升泛化与概率校准
    

```
y_true = 1        → 1 - ε
y_false = 0       → ε / (K - 1)

```

适合场景：

- 数据噪声
    
- 类间边界模糊（如睁/闭眼）
    

---

### 4.2.4 检测任务损失

- 分类：CE / Focal Loss
    
- 回归：L1 / GIoU / CIoU
    
- 目标性：BCE
    

---

# 5 优化器（Optimizer）

## 5.1 常用优化器

### 5.1.1 SGD

- 优点：泛化好
    
- 缺点：收敛慢，参数敏感
    

适合：

- 大数据
    
- 检测任务
    

---

### 5.1.2 Adam / AdamW（工程首选）

- 自适应学习率
    
- 对初始 lr 不敏感
    
- AdamW 正确处理 weight decay
    

适合：

- 分类
    
- 迁移学习
    
- 小数据
    


---

# 6 训练参数（Training Configuration）

## 6.1 学习率（LR）

- backbone lr < head lr（迁移学习）
    
- warmup 非常重要
    

示例：

```
backbone_lr: 1e-4
head_lr:     1e-3
```

---

## 6.2 Batch Size

- 与显存强相关
    
- 与学习率强耦合
    

经验法则：

`batch × lr ≈ 常数`

---

## 6.3 Epoch / Iter

- 分类：20–100 epochs
    
- 微调：5–30 epochs
    
- 观察 loss 是否真正下降，而非 epoch 数字
    

---

## 6.4 正则化

- Weight Decay
    
- Dropout（轻量模型慎用）
    
- 数据增强（优先级最高）
    

---

# 7 迁移学习（Transfer Learning）

## 7.1 两种主流工程路线

### 7.1.1 路线 A：冻结 backbone，训练 head（首选）

`Backbone（frozen） ↓ Head（trainable）`

- 快
    
- 稳
    
- 易部署
    

---

### 7.1.2 路线 B：逐步解冻（进阶）

`Head → 高层 Backbone → 全模型`

适合：

- 域差异较大
    
- 数据量足够
    

---

## 7.2 分布对齐（理论为主）

- MMD / CORAL / DANN
    
- 工程复杂
    
- 部署困难
    

⚠️ 一般不作为量产首选

---
