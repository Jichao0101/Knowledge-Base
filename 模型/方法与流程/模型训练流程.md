# 1 整体工程视角

一个可量产的深度学习项目，必须明确分层：

```
数据层
  ↓
前处理（Preprocessing）
  ↓
模型结构（Model + forward）
  ↓
损失函数（Loss）
  ↓
优化器（Optimizer）
  ↓
训练策略（Scheduler / AMP / EMA / DDP）
  ↓
.pth（参数快照）
  ↓
ONNX（静态计算图）
  ↓
HBM / TensorRT（硬件执行图）
  ↓
SDK（前处理 + 推理 + 后处理 + 业务逻辑）


```

**关键原则：**

- `.pth` 不是模型，只是参数
    
- **模型 = 结构代码 + forward 逻辑**
    
- ONNX / HBM 需要**静态算子图**
    

___

# 2 数据与预处理（Preprocessing）

##  预处理的工程定位

> **预处理不是“辅助步骤”，而是模型的一部分（但不在模型图里）**

训练 / 导出 / SDK **必须一致**。

---

##  常见视觉预处理流程

### 输入数据来源

- Camera / Video / Image
    
- 格式：BGR / RGB / NV12 / YUV420
    

---

###  颜色空间转换

`NV12 / YUV → RGB（或 BGR）`

⚠️ 错误的颜色空间 = 模型性能直接下降  
（这是工程中最常见的“模型没效果”原因之一）

---

###  Resize / Crop / Letterbox

- 分类模型：  
    `Resize → CenterCrop / ROI Crop`
    
- 检测模型：  
    `Resize + Padding (letterbox)`
    

⚠️ **训练与推理必须使用同一种几何变换**

---

###  Normalize

典型 ImageNet 归一化：

```
mean = (0.485, 0.456, 0.406) 
std  = (0.229, 0.224, 0.225)
```

工程原则：

- Normalize **永远不进模型**
    
- 但 **必须被写进“模型规范文档”**
    

---

###  Tensor 格式

`HWC → CHW uint8 → float32 / float16`

---

##  预处理一致性原则

> **如果训练前处理 ≠ SDK 前处理，  
> 模型表现没有任何讨论价值**

---

# 3 模型结构与计算图（Model & Graph）

## 3.1 模型结构的工程定义

`Backbone + Neck（可选） + Head`

- Backbone：特征提取（ResNet / MobileNet / CSP）
    
- Head：
    
    - 分类：Linear / Conv + Pool
        
    - 检测：BBox + Class + Objectness
        

---

## 3.2 计算图与算子拓扑

- 模型在 PyTorch 中是 **动态图**
    
- 导出 ONNX 时，**一次 forward 会被冻结成静态图**
    

**工程约束：**

- ❌ Python if / for 控制结构（依赖数据）
    
- ❌ shape 依赖 runtime tensor
    
- ✅ 所有逻辑必须转为 tensor 运算
    

---

## 3.3 dummy input 的工程意义

```
dummy = torch.randn(1, 3, H, W) 
torch.onnx.export(model, dummy, ...)

```
dummy input 用来：

- 固定输入 shape
    
- 决定 forward 路径
    
- 生成完整算子图
    

📌 **模型结构不应依赖输入内容，只应依赖输入 shape**

---

# 4 损失函数（Loss Function）

## 4.1 工程视角下的损失函数本质

> **损失函数不是“算错了多少”，  
> 而是“如何惩罚错误，尤其是自信的错误”。**

---

## 4.2 分类任务：交叉熵（Cross Entropy）

### 4.2.1 多分类交叉熵

$$\mathcal{L} = -\log p_{\text{true}}$$​

工程含义：

- 预测正确且自信 → loss → 0
    
- 预测错误且自信 → loss → ∞（理论上）
    

---

### 4.2.2 梯度的关键性质

softmax + CE：

$$\frac{\partial \mathcal{L}}{\partial z_k} = p_k - y_k$$​

工程意义：

- 错得越自信，梯度越大
    
- 训练稳定、收敛快
    
- 非常适合分类任务
    

---

### 4.2.3 Label Smoothing

目的：

- 削弱“无限确信”
    
- 提升泛化与概率校准
    

```
y_true = 1        → 1 - ε
y_false = 0       → ε / (K - 1)

```

适合场景：

- 数据噪声
    
- 类间边界模糊（如睁/闭眼）
    

---

### 4.2.4 检测任务损失

- 分类：CE / Focal Loss
    
- 回归：L1 / GIoU / CIoU
    
- 目标性：BCE
    

---

# 5 优化器（Optimizer）

## 5.1 常用优化器

### 5.1.1 SGD

- 优点：泛化好
    
- 缺点：收敛慢，参数敏感
    

适合：

- 大数据
    
- 检测任务
    

---

### 5.1.2 Adam / AdamW（工程首选）

- 自适应学习率
    
- 对初始 lr 不敏感
    
- AdamW 正确处理 weight decay
    

适合：

- 分类
    
- 迁移学习
    
- 小数据
    


---

# 6 训练参数（Training Configuration）

## 6.1 学习率（LR）

- backbone lr < head lr（迁移学习）
    
- warmup 非常重要
    

示例：

```
backbone_lr: 1e-4
head_lr:     1e-3
```

---

## 6.2 Batch Size

- 与显存强相关
    
- 与学习率强耦合
    

经验法则：

`batch × lr ≈ 常数`

---

## 6.3 Epoch / Iter

- 分类：20–100 epochs
    
- 微调：5–30 epochs
    
- 观察 loss 是否真正下降，而非 epoch 数字
    

---

## 6.4 正则化

- Weight Decay
    
- Dropout（轻量模型慎用）

- Label Smoothing
    
- 数据增强（优先级最高）
    

---

# 7 迁移学习（Transfer Learning）

## 7.1 两种主流工程路线

### 7.1.1 路线 A：冻结 backbone，训练 head（首选）

`Backbone（frozen） ↓ Head（trainable）`

- 快
    
- 稳
    
- 易部署
    

---

### 7.1.2 路线 B：逐步解冻（进阶）

`Head → 高层 Backbone → 全模型`

适合：

- 域差异较大
    
- 数据量足够
    

---

## 7.2 分布对齐（理论为主）

- MMD / CORAL / DANN
    
- 工程复杂
    
- 部署困难
    

⚠️ 一般不作为量产首选

# 9 训练动力学（Training Dynamics）

> 训练动力学描述的是：  
> **模型参数如何在损失函数与优化器的驱动下，沿着时间（iteration / epoch）演化。**

理解训练动力学，才能：

- 判断训练是否“真的在学”
    
- 分辨“指标抖动 / 假收敛 / 过拟合”
    
- 合理使用 AMP、EMA、DDP 等工程技巧
    

---

##  训练的本质：参数在高维空间中的轨迹

###  参数空间视角

- 模型参数：
    
    $\theta \in \mathbb{R}$
- 损失函数定义了一个 **高维能量面**：
    
    $\mathcal{L}(\theta)$

训练过程本质是：

`在高维非凸空间中 沿着梯度方向 寻找低能量区域`

⚠️ 不是“找到最优点”，而是：

- 找到一个 **泛化良好的盆地（flat minimum）**
    

---

## 10.2 梯度、噪声与泛化

### 10.2.1 梯度更新的真实形式

以 SGD / Adam 为例：

$\theta{t+1}=\theta_t−η⋅(∇L(\theta_t)+\epsilon_t)$

其中：

- $\eta$：学习率
    
- $\epsilon_t$​：**由 mini-batch 引入的随机噪声**
    

### 工程意义

- 适量噪声有助于：
    
    - 跳出尖锐极小值
        
    - 提升泛化能力
        

📌 **batch size、lr、数据增强，本质都在调控“噪声尺度”**

---

## 10.3 学习率（Learning Rate）的动力学角色

### 10.3.1 学习率不是“速度”，而是“搜索尺度”

- lr 太大：
    
    - 震荡
        
    - 不收敛
        
- lr 太小：
    
    - 收敛慢
        
    - 易陷入局部尖锐最小值
        

---

### 10.3.2 Cosine Annealing 的动力学意义

`初期：大步探索 中期：稳定下降 后期：细粒度收敛`

数学形式（简化）：

$\eta_t = \eta_{min} + \frac{1}{2}(\eta_{max} - \eta_{min})(1 + \cos(\pi t / T))$

工程效果：

- 减少后期震荡
    
- 提升最终收敛质量
    
- 常用于分类任务
    

---

### 10.3.3 Warmup 的真实用途

Warmup 不是为了“好看”，而是为了解决：

- 初期梯度分布极不稳定
    
- BatchNorm 统计量尚未可靠
    
- 预训练权重被大梯度破坏
    

📌 **Warmup 是工程稳定性工具，而不是调参技巧**

---

## 10.4 Batch Size × Learning Rate 的耦合关系

### 10.4.1 经验法则的来源

常见经验：

`batch × lr ≈ 常数`

原因：

- 更大的 batch → 梯度噪声更小
    
- 噪声变小 → 需要更大的 lr 才能保持探索能力
    

---

### 10.4.2 工程取舍

- 小 batch：
    
    - 噪声大
        
    - 泛化好
        
    - 不稳定
        
- 大 batch：
    
    - 稳定
        
    - 需要 lr 放大
        
    - 易过拟合
        

---

## 10.5 正则化在动力学中的作用

### 10.5.1 Weight Decay（L2 正则）

$\mathcal{L}_{total} = \mathcal{L} + \lambda \|\theta\|^2$

动力学效果：

- 抑制参数爆炸
    
- 偏向平坦解
    
- 提升泛化
    

📌 AdamW 的关键点：  
**weight decay 不参与动量计算**

---

### 10.5.2 Label Smoothing 的动力学影响

- 减小极端梯度
    
- 避免 logits 无限放大
    
- 改善概率校准
    

工程效果：

- loss 更平滑
    
- 梯度更稳定
    
- 对噪声标签更鲁棒
    

---

## 10.6 数据增强：最强的隐式正则

### 10.6.1 数据增强如何影响训练动力学

从优化角度看：

`每一次增强 = 改变损失函数形状`

效果：

- 扩大有效数据分布
    
- 增加梯度多样性
    
- 降低过拟合风险
    

📌 数据增强优先级 **高于任何正则项**

---

## 10.7 AMP（混合精度）的动力学影响

### 10.7.1 AMP 做了什么

- 前向 / 反向：FP16
    
- 参数更新：FP32
    
- 使用 loss scaling 防止 underflow
    

### 对训练动力学的影响

- 数值噪声略增（通常可接受）
    
- 梯度分辨率降低
    
- 实践中：
    
    - 收敛路径基本一致
        
    - 最终性能接近 FP32
        

📌 **AMP 是工程加速手段，不是算法改变**

---

## 10.8 EMA（指数滑动平均）的动力学意义

### 10.8.1 EMA 的定义

$\theta^{EMA}_t = \alpha \theta^{EMA}_{t-1} + (1-\alpha)\theta_t$

### 工程解释

- 训练参数在震荡
    
- EMA 对轨迹做低通滤波
    
- 结果更接近“稳定中心”
    

---

### 10.8.2 EMA 适合的场景

- 小模型
    
- 小数据
    
- 分类任务
    
- 推理对稳定性敏感
    

📌 EMA **不改变训练过程**，只改变“使用哪个权重做评估 / 导出”

---

## 10.9 DDP（分布式训练）下的动力学一致性

### 10.9.1 DDP 的核心机制

```
每卡前向/反向
↓
AllReduce 梯度
↓
参数同步更新

```

数学上等价于：

`更大的 batch size`

---

### 10.9.2 动力学影响

- 梯度噪声降低
    
- 收敛更稳定
    
- 可能需要：
    
    - lr scaling
        
    - 更强正则
        

---

## 10.10 如何判断训练是否“健康”

### 10.10.1 健康训练的典型特征

- loss 平滑下降
    
- train / val gap 可控
    
- Macro-F1 稳定提升
    
- EMA ≥ raw model
    

---

### 10.10.2 常见异常模式

|现象|可能原因|
|---|---|
|loss 不降|lr 太大 / 前处理错误|
|train 好 val 差|过拟合 / 数据泄漏|
|指标抖动|batch 太小 / lr 不稳|
|EMA 显著更好|模型震荡较大|

---

## 10.11 工程结论（训练动力学共识）

- 训练不是“跑参数”，而是**控制动力系统**
    
- lr、batch、增强、正则 **共同决定轨迹**
    
- AMP / EMA / DDP 是 **轨迹稳定器**
    
- 泛化来自：
    
    - 噪声
        
    - 平坦解
        
    - 一致的工程约束
