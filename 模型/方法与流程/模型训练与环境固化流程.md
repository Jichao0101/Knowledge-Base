
# 1 核心原则

## 1.1 分层原则（

```
系统层（OS / CUDA / Driver） 
↓ 
运行时层（conda Python 环境） 
↓ 
项目层（代码 + Python 依赖）

```

---

## 1.2 角色分工总结

|层级|工具|解决什么|
|---|---|---|
|系统层|Dockerfile|能不能跑、跑在哪|
|运行时层|conda / environment.yml|Python 版本 + 二进制依赖|
|项目层|requirements.txt / pip|代码 import 依赖|

---

# 2 标准模型训练流程

## 2.1 总览

```
阶段 1：本地 conda 环境（开发） 
阶段 2：冻结 Python 环境 
阶段 3：Docker 固化环境 
阶段 4：Docker 中正式训练 / 复现
```

---

## 2.2 阶段 1：本地 conda 环境训练（开发阶段）

### 2.2.1 目标

- 快速迭代
    
- 快速 debug
    
- 验证：
    
    - 数据 pipeline
        
    - 模型结构
        
    - loss / 收敛性
        
    - 训练逻辑是否正确
        

> **任何不能在本地 conda 跑通的训练，不允许进入 Docker 阶段**

---

### 2.2.2 本地 conda 环境创建


📌 此阶段特点：

- 允许快速试错
    
- 允许版本微调
    
- 不要求绝对可复现
    

[[Conda使用指南]]

---

## 2.3 阶段 2：冻结环境

### 2.3.1 触发条件（必须全部满足）

- 训练代码稳定
    
- 数据无明显问题
    
- 指标达到预期
    
- 配置文件基本确定
    

---

### 2.3.2 冻结方式

#### 2.3.2.1 方式 A：environment.yml（推荐）

`conda env export > environment.yml`

优点：

- 可读性强
    
- 适合团队协作
    

---

#### 2.3.2.2 方式 B：显式 lock（最严格）

`conda list --explicit > conda-lock.txt`

优点：

- 100% 复现  
    缺点：
    
- 平台相关（Linux / CUDA 绑定）
    

---

### 2.3.3 冻结文件的职责

> 冻结文件的作用是：  
> **“记录那一刻能跑通训练的依赖解”**

---

## 2.4 阶段 3：Dockerfile 固化系统环境

### 2.4.1 Dockerfile 的职责

Dockerfile **只负责**：

- OS
    
- CUDA / cuDNN
    
- 系统库
    
- conda 安装
    

Dockerfile **不负责**：

- 项目 Python 依赖细节
    
- 训练逻辑
    

---

### 2.4.2 标准训练 Dockerfile 模板

```
FROM nvidia/cuda:11.8.0-cudnn8-runtime-ubuntu22.04

# 系统依赖
RUN apt-get update && apt-get install -y \
    git wget curl \
    libgl1 libglib2.0-0 ffmpeg \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

# 安装 Miniconda
ENV CONDA_DIR=/opt/conda
RUN wget -q https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh && \
    bash Miniconda3-latest-Linux-x86_64.sh -b -p ${CONDA_DIR}

ENV PATH=${CONDA_DIR}/bin:$PATH
SHELL ["/bin/bash", "-c"]

WORKDIR /workspace

```

---

## 2.5 阶段 4：Docker 中复现 conda 环境并训练

### 2.5.1 构建 conda 环境（基于冻结文件）

```
COPY environment.yml /workspace/

RUN conda env create -f environment.yml

```

激活环境（示例）：

`RUN echo "source activate train" >> ~/.bashrc`

---

### 2.5.2 拷贝代码并启动训练

```
COPY . /workspace/

CMD ["bash", "-c", "source activate train && python tools/train.py"]

```

## 2.7 工程级 Checklist

### 2.7.1 进入 Docker 前必须满足

-  本地 conda 能完整跑通训练
    
-  数据路径、增强逻辑已确认
    
-  指标稳定
    
-  environment.yml 已生成
    

---

### 2.7.2 Docker 训练阶段必须满足

-  Dockerfile 不包含业务 pip 依赖
    
-  conda 环境来自冻结文件
    
-  CUDA 版本明确
    
-  训练日志可复现
    

---

## 2.8 一句话工程总结

> **模型训练应先在本地 conda 环境完成验证，再通过 Docker 固化环境进行正式训练与复现。  
> conda 负责 Python 运行时，Docker 负责系统与算力边界，二者职责不可混淆。**
