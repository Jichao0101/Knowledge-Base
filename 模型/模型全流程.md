# 3 模型生命周期总览

```
数据准备
  ↓
模型训练（FP32）
  ↓
pth（权重）
  ↓
ONNX（静态计算图）
  ↓
量化（PTQ / QAT）
  ↓
编译（BC）
  ↓
部署模型（HBM）
  ↓
SDK 推理 + 业务逻辑

```
    
- ONNX / BC / HBM 必须是 **静态算子图**

# 4 顶层工程公理（必须无条件满足）

以下原则具有最高优先级，**任何阶段不得违反**：

## 4.1 输入语义等效一致性

- 任意模型形态下
    
- 进入第一层算子的输入张量
    
- 在**语义上必须等效**
    

详见：

- [[输入语义一致性]]
    

---

## 4.2 统一评估与 Gate 放行原则

- PyTorch FP32 是唯一 Golden Baseline
    
- 跨阶段对比必须满足“五同一”
    
- 每一阶段必须通过 Gate 才能进入下一阶段
    

详见：

- [[模型评估与部署指南]]

# 5 阶段划分与责任边界

## 5.1 阶段 A：模型训练与基线建立

**目标**

- 训练模型
    
- 建立 **FP32 精度基线（Golden Baseline）**
    

**输入**

- 训练数据集
    
- 训练配置
    

**输出（必须）**

- `.pth` 权重
    
- 基线评估结果（FP32）
    

**参考文档**

- [[模型训练流程]]
    
- [[模型训练与环境固化流程]]
    

---

## 5.2 阶段 B：ONNX 导出与一致性验证

**目标**

- 将动态图冻结为静态计算图
    
- 验证 **导出本身不引入精度问题**
    

**输入**

- `.pth`
    
- dummy input
    

**输出（必须）**

- `.onnx`
    
- ONNX 推理评估结果
    

**Gate 要求**

- 与 FP32 基线几乎一致
    

**参考文档**

- [[模型评估与部署指南]]
    

---

## 5.3 阶段 C：量化与编译验证

**目标**

- 引入量化与编译
    
- 验证数值近似与前处理入图的综合影响是否可控
    

**输入**

- `.onnx`
    
- calibration 数据集
    

**输出（必须）**

- `.bc`
    
- 量化/编译态评估结果
    

**Gate 要求**

- 精度下降在可接受阈值内
    
- 不出现系统性 Recall 崩塌
    

**参考文档**

- [[模型量化]]
    
- [[模型评估与部署指南]]
    

---

## 5.4 阶段 D：最终部署验证

**目标**

- 验证真实部署态模型是否满足业务要求
    

**输入**

- `.hbm`
    
- 真实输入路径（如 NV12）
    

**输出（必须）**

- 部署态精度评估
    
- 性能评估（latency / FPS / 内存）
    

**最终裁决原则**

> **模型是否可上线，只看 Stage D**

**参考文档**

- [[模型评估与部署指南]]
    
- [[J6M模型部署流程]]

# 7 云平台训练

当模型训练规模或算力需求提升时，可引入云平台训练。

- 算力与环境解耦
    
- 训练过程可复现
    
- 产物可回流到本流程
