# 1 目标与适用范围

**模型全流程中的最高公理之一**：

> **无论模型处于何种形态，进入第一层算子的输入张量在“语义上必须等效”。**

适用于：

- 模型训练
    
- ONNX 导出
    
- PTQ / QAT 校准
    
- 编译与上板部署
    

---

# 2 什么是“输入语义等效”

输入语义等效 ≠ 实现方式一致。

允许不同实现位置：

- 图外（Python / SDK / CPU）
    
- 图内（ONNX folding / NPU 前处理）
    

但要求：

- **同一物理图像内容**
    
- **同一几何映射规则**
    
- **同一数值归一化定义**
    

---

# 3 必须等效的维度

以下维度必须等效：

- shape（NCHW / NHWC）
    
- dtype（float32 / float16）
    
- 数值范围（scale / norm）
    
- 通道顺序（RGB / BGR）
    
- 空间映射（resize / letterbox / padding）
    

---

# 4 常见破坏等效性的错误

- 颜色空间不一致（RGB ↔ BGR ↔ NV12）
    
- resize / letterbox 策略不同
    
- padding 回映射错误
    
- normalize 写在不同位置但参数不一致
    

---

# 5 验证方法

推荐使用：

- Regression Set 逐图对齐
    
- 第一层输入 tensor dump 对比
    
- bbox IoU / score 分布对齐
    

---

# 6 工程结论

> **一旦输入语义不等效，任何精度对比都不成立。**
