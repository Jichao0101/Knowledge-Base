# 1 模型训练

## 1.1 BatchNorm

$\hat{x}=\frac{x-\mu}{\sigma}$通过标准化-控制激活的均值和方差，避免梯度爆炸 / 消失，但是会导致

- 每一层输出 **被强制成零均值、单位方差**
    
- 网络 **失去自由选择特征尺度和偏置的能力**

$y= \gamma \hat{x}+\beta$ **让网络在“数值稳定”的前提下，自由选择“想要的分布”**

- 如果模型不想 BN 影响它  
    → 学到 `γ ≈ σ, β ≈ μ`
    
- 如果模型想压缩某通道  
    → γ 变小
    
- 如果模型想整体偏移  
    → β ≠ 0
    
 **BN = 稳定化 + 可学习的仿射重参数化**
 
$running_mean←(1−momentum)⋅running_{mean}+ momentum * \mu_{batch}$

训练时维护 running_mean / running_var，并在 eval() 时使用它们

# 2 硬件加速

## 2.1 VAE和TAE

| 维度 | VAE（Vector Acceleration Engine） | TAE（Tensor Acceleration Engine） |
|----|----|----|
| 面向算子 | Element-wise / 向量算子 | 卷积 / GEMM / MatMul |
| 计算基本单元 | 标量 / SIMD 向量 | MAC 阵列 / Systolic Array |
| 数据复用 | 低（元素用一次或少量） | 高（输入/权重多次复用） |
| 计算密度 | 低 | 极高 |
| 性能瓶颈 | 内存带宽 | 数据搬运与复用调度 |
| 数据流模型 | load → op → store | tile load → 本地缓存 → MAC 累加 → store |
| 是否保留中间状态 | 否（即算即写） | 是（局部累加寄存） |
| 算子耦合度 | 弱 | 强 |
| 典型硬件结构 | ALU / SIMD | Tensor Core / MAC Array |
| 适合算子 | ReLU / Add / Mul / BN / Scale | Conv / FC / Attention |


# 3 模型转换

## 3.1 格式对比
| 维度 | ONNX | BC | HBM |
|----|----|----|----|
| 层级 | 模型语义层 | 执行计划层 | 设备程序层 |
| 是否可跨平台 | 是 | 否（芯片相关） | 否（设备相关） |
| 是否可改图 | 否 | 是（合法重写） | 否 |
| 是否含 preprocess | 通常不含 | 可融合 | 已固化 |
| 是否考虑硬件 | 否 | 是 | 是（极致） |
| 可读性 | 高 | 中 | 极低 |
| 用途 | 表达 / 验证 | 编译 / 优化 | 运行 |

## 3.2 为什么 ONNX → BC 的校准集取自训练集（或高度一致分布）

量化校准的目标是让量化后的模型尽可能逼近浮点模型在其**工作分布**上的行为：

- 使用训练分布（或高度一致的数据）能使量化参数在“模型最熟悉的区域”误差最小
    
- 从工程角度降低部署态分布内的系统性掉点风险
    

> ✅ 校准集不是“评估集”，而是“让量化更贴近业务分布的工具集”。
