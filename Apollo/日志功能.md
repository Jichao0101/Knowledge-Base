
Apollo使用google的的轻量日志库glog，支持常用的日志级别，自定义级别，条件输出，DEBUG日志等，同时支持gflag的参数配置方式。
# 1 Glog库
## 1.1 日志级别
日志级别有4种，INFO WARNING ERROR FATAL，数值越大级别越高
```
const int GLOG_INFO = 0, GLOG_WARNING = 1, GLOG_ERROR = 2, GLOG_FATAL = 3, NUM_SEVERITIES = 4;
const int INFO = GLOG_INFO, WARNING = GLOG_WARNING, ERROR = GLOG_ERROR, FATAL = GLOG_FATAL;
```

不同的日志级别会输出到不同的文件，与日志级别一样，文件名后缀共有4个，分别是.INFO .WARNING .ERROR .FATAL。X级别的日志会输出到所有<=X的日志文件里，例如error日志在.INFO .WARNING .ERROR日志里都会输出
## 1.2 输出信息的结构 
log类型	log日期	log时间	进程PID	输出信息
IWEF	月日信息	时间信息	所在文件名称和行数	调试信息
## 1.3 条件输出
当某些变量满足某些条件时才输出日志
LOG_IF(INFO, condition); //当condition为true时输出日志。
LOG_EVERY_N(INFO, N); //当运行该语句时，每隔N次输出日志，
LOG_IF_EVERY_N(INFO, condition, N); //当满足condition时，每N次输出日志。
LOG_FIRST_N(INFO, N); //此语句的前N次输出到日志，google::COUNTER记录语句的执行次数。

## 1.4 配置
glog的选项可以通过gflags配置，常用flag有
- logtostderr: bool型，设置为true后日志不输出到文件，在屏幕打印，默认值为false。
- stderrthreshold： int型，>=该值的日志级别除了输出到文件，同时在屏幕打印，默认值为2，即屏幕和日志同时输出ERROR FATAL错误。
- minloglevel: int型，>=该值的日志级别才会输出到文件，默认值为0
```
#使用系统变量设置
GLOG_minloglevel=3 ./application 
```

Orin中使用了DECLARE_int32(minloglevel)命名gflags变量，因此可以在启动程序时在命令行配置最低日志等级
```
./application --minloglevel=1 
```

- log_dir: string型，日志输出位置，默认为”“。如果在程序里手动修改FLAGS_log_dir需要在调用google::InitGoogleLogging之前使用
- v:自定义日志级别
## 1.5 debug日志
 glog提供特定的宏只在debug模式下生效。以下分别对应LOG、LOG_IF、DLOG_EVERY_N操作的专用宏。
```
DLOG(INFO) << "Found cookies";
DLOG_IF(INFO, num_cookies > 10) << "Got lots of cookies";  DLOG_EVERY_N(INFO, 10) << "Got the " << COUNTER << "th cookie";
```

## 1.6 自定义日志
glog提供了VLOG宏用于自定义日志的输出，用户可以使用–v=数字来指定自定义的日志级别。
与系统日志级别不同的是，vlog数值越低，级别越高。例如VLOG(4)定义该日志为4级，如果-v=3则该日志不输出，如果-v=4则输出该日志
