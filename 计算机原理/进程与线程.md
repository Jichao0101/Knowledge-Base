# 1 基本概念

- **进程（Process）**  
  操作系统中进行**资源分配**的基本单位。
- **线程（Thread）**  
  操作系统中进行**调度执行**的基本单位。
- **多线程（Multithreading）**  
  一个进程中同时存在多个线程并发执行的运行方式。

> **一个进程 = 一堆资源 + 至少一个执行流（线程）**  
> **多线程 = 在同一堆资源里跑好几个执行流**

---

# 2 进程（Process）

## 2.1 进程的资源

每个进程一般拥有自己**独立**的：

- **虚拟地址空间**
  - 代码段（text）
  - 数据段（data / bss）
  - 堆（heap）
  - 若干个**栈**（stack，给不同线程使用）
- **打开的文件**
  - 文件描述符表 / 句柄表（如 `stdin/stdout/stderr`、socket 等）
- **全局变量所在的内存**
  - 全局 / 静态变量归属于进程的地址空间，作为共享的可读写状态
- **其他进程级资源**
  - 信号处理方式
  - 进程 ID、父子关系
  - 权限信息等

## 2.2 进程的特点

- 拥有**独立的地址空间**，不同进程之间**默认内存隔离**；
- 进程间通信（IPC）相对开销较大，如：
  - 管道（pipe）
  - 消息队列
  - 共享内存
  - socket 等
- 创建 / 销毁成本相对线程更高（涉及页表、文件表等结构）。

# 3 线程（Thread）

## 3.1 线程的共享

同一进程内的多个线程**共享**：

- 进程的**虚拟地址空间**
  - 代码段、全局变量、堆
- 进程打开的文件（文件描述符 / 句柄）
- 进程的其他部分资源（信号、部分内核对象等）

所以在同一进程内：

> 一个线程 `malloc` 出来的内存，另一个线程可以直接用；  
> 一个线程打开的文件，其他线程可以直接 `read/write`。

只要某个对象仍然在其生命周期内，并且其地址被传递给多个线程，那么这些线程都可以通过该地址访问这个对象——无论这个对象实际存储在栈上、堆上还是全局/静态区。  
但需要额外保证：

1. 对象的生命周期覆盖所有访问它的线程（避免悬空指针）；
    
2. 多线程读写时使用适当的同步手段（避免 data race）。

**“共享” = 同一地址空间 + 正确的指针/引用 + 合法的生命周期 + 正确的同步**

## 3.2 线程的私有

每个线程有自己**私有**的：

- **寄存器现场**
  - 通用寄存器
  - 程序计数器（PC，Program Counter）
  - 部分状态寄存器等
- **线程栈（stack）**
  - 用来保存函数调用栈帧、局部变量、返回地址等
- **线程 ID**
- **线程局部存储（TLS，Thread Local Storage）**
  - 如 `thread_local` / `__thread` 变量等

## 3.3 线程不是函数

线程本身只是**执行流**，不是“函数”。

在实际 API 中，一般是这样使用的：

- 创建线程时，传入一个**线程入口函数的指针**（加可选参数）；
- 操作系统（或运行库）为线程分配栈和 TCB（线程控制块）；
- 新线程从这个入口函数开始执行，直到函数返回或调用结束 API。

伪代码示意：

```c
// 线程入口函数
void* worker(void* arg) {
    // do something
    return NULL;
}

// 创建线程
pthread_t tid;
pthread_create(&tid, NULL, worker, arg);
```

# 4 多线程（Multithreading）
## 4.1 多线程

在同一个进程内部，创建多个线程，这些线程共享进程的资源，但能在不同 CPU 核心上并发/并行执行。

典型用途：

- IO 密集型：隐藏 IO 等待，提升吞吐（比如网络服务器、日志系统等）

- CPU 密集型：利用多核 CPU 并行计算（如图像处理、数值计算等）

## 4.2 多线程带来的问题

**多线程的核心矛盾**：

共享带来通信便利，也带来数据竞争与同步问题。

**常见问题**：

- 竞态条件（race condition）

	多个线程同时读写同一共享变量，结果依赖执行顺序

- 数据不一致

	没有正确加锁、使用原子操作，导致可见性问题

- 死锁（deadlock）

	多线程循环等待对方持有的锁，互相卡死

- 假唤醒、优先级反转 等更细节的问题

**常见同步手段**：

- 互斥锁（mutex）

- 读写锁（rwlock）

- 条件变量（condition variable）

- 信号量（semaphore）

- 原子操作（atomic）

- 屏障（barrier）

# 5 进程 vs 线程：对比表

| 对比项     | 进程                    | 线程                         |
| ------- | --------------------- | -------------------------- |
| 定义      | 资源分配的基本单位             | 调度执行的基本单位                  |
| 地址空间    | 各进程相互独立               | 同一进程内的线程共享同一地址空间           |
| 资源      | 拥有独立的文件表、全局变量等资源      | 共享进程内资源，仅有栈、寄存器等少量私有资源     |
| 创建/销毁开销 | 大（需要建立/销毁地址空间、文件表等）   | 小（主要分配栈和线程控制块等）            |
| 通信方式    | IPC:管道、队列、共享内存、socket | 直接读写共享内存                   |
| 故障影响范围  | 一个进程崩溃，一般不影响其他独立进程    | 一个线程崩溃，可能拖垮整个进程（如非法访问共享内存） |
| 调度粒度    | 粗，适合不同应用之间隔离          | 细，适合同一程序内的并发任务             |
| 使用场景    | 服务隔离，多任务并行，不同权限/安全上下文 | 高并发服务器，多核计算，UI+worker线程    |
