# 1 栈内存机制

## 1.1 定义与用途

栈是编译器用于管理函数调用过程的一块线性内存区域，用于存放
- 函数局部变量
- 函数返回地址
- 保存的寄存器状态
- 函数参数（视调用约定而定）

每次函数调用都会创建一个**独立的栈帧**

## 1.2 栈帧布局的确定时机

栈帧布局（包括局部变量大小与地址偏移）在**编译期**即可确定
```
void foo() {
    int a;      // 4 bytes
    double b;   // 8 bytes
}
```
编译器在生成汇编代码之前，就已确定变量的大小和在栈帧中的偏移

| 变量  | 大小  | 在栈帧中的偏移  |
| --- | --- | -------- |
| a   | 4B  | [rbp-4]  |
| b   | 8B  | [rbp-12] |

> [!NOTE] 
>栈变量地址偏移固定、布局固定，因此在运行时无法修改变量大小

## 1.3 栈内存的分配与回收

栈的内存管理不需要操作系统参与，仅通过CPU指令移动栈指针完成，时间复杂度`O(1)`,效率非常高
```
sub rsp, 16; 分配栈空间
add rsp, 16; 回收栈空间
```
## 1.4 栈的限制

- 大小受限（一般MB级别）
- 变量生命周期受限于函数作用域
- 变量空间不能在运行时动态扩展

# 2 堆内存机制

## 2.1 定义与用途

堆是程序运行中用于动态分配内存的区域，适用于
- 生命周期超出函数作用域的数据
- 无法在编译器确定大小的数据结构（vector、链表等）
-
## 2.2 堆内存分配方式

通过运行时函数申请
```
int* p = malloc(sizeof(int) * n);
auto p = new int[n];
```

堆管理涉及：
- 空闲块查找
- 内存标记与释放
- 内存碎片整理
- 在必要时向系统请求更多堆空间

堆的分配/释放可能涉及内核，时间复杂度不定（受分配器影响），效率明显低于栈

## 2.3 堆栈对比


| 对比项    | 栈          | 堆         |
| ------ | ---------- | --------- |
| 大小确定时机 | 编译器确定      | 运行期确定     |
| 分配方式   | 调整栈指针      | 动态内存管理    |
| 生命周期   | 函数作用域      | 手动管理或垃圾回收 |
| 系统调用   | 无          | 可能需要      |
| 性能     | 极高         | 相对较低      |
| 典型用途   | 局部变量、控制流数据 | 大型对象、动态数组 |
