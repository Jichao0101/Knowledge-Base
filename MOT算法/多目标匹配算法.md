# 1 二分图最大匹配-匈牙利算法
## 1.1 概念
1. 二分图：若无向图G=(V,E)的顶点V能够划分为两个交集为空的顶点集，且任意边的两个端点分属于两个集合，则称图G为一个二分图
2. 匹配：在G的一个子图M中，M的边集中任意两条边都不依附于同一个顶点，则称M是一个匹配
3. 交替路：从一个未匹配点出发，一次经过非匹配边、匹配边、非匹配边形成的路径为交替路
4. 增广路：从一个未匹配点出发，走交替路，若能到达另一个为匹配点，则这条交替路称为增广路，即把这条路上的匹配状态取反，则匹配规模+1
5. 邻接表存图
	1. 边e[i] ： 存第i条出边，v代表边的终点，ne代表从同一起点出发的下一条边在数组e中的index
	2. h[i] : 存起点i的第一条出边的index
```
	struct edge {
	int v, ne;
	} e[M];
```
## 1.2 算法实现
### 1.2.1 算法流程
1. DFS函数：实现寻找增广路
	- 对于左边顶点u，遍历其所有出边到右边顶点v
	- 如果v未被访问，标记为已访问
	- 如果v未匹配，或者从v的匹配顶点出发能够找到增广路，则匹配u和v，并返回true
2. 主循环：对每个左边顶点，调用DFS尝试匹配
	- 每次调用前重置访问数组
	- 如果DFS返回true，匹配数加1


# 2 二分图最大权匹配-KM算法
## 2.1 概念
1. 可行顶标 feasible labeling：给每个节点分配一个权值lx[x], ly[y], 对于所有边(x,y)满足lx[x] + ly[y] >= w(x, y)
2. 相等子图：在一组可行顶标下原图的生成子图，包含所有点但只包含满足lx[x] + ly[y] == w(x, y)的边。匈牙利算法可以沿着任意存在的边寻找增广路，但是KM算法必须在相等子图中寻找增广路
3. 定理：如果相等子图存在完美匹配，则该匹配就是二分图的最大权完美匹配
## 2.2 算法实现
### 2.2.1 算法原理
原问题
$$
\max\sum_{i}\sum_{j}w_{ij}x_{ij}
$$
约束条件
$$
\begin{aligned}
\sum_{i}w_{ij}=1\\
\sum_{j}w_{ij}=1\\
x_{ij}\geq 0
\end{aligned}
$$
即原问题为在边上挑选，最大化权重和。
对偶问题为
$$
\min\sum_{i}u_i+\sum_jv_j
$$
约束条件：
$$
u_i+v_i\geq w_ij
$$
即对偶问题为给每个顶点分配一个“顶标” $u_i,v_j$​，使得它们的和至少覆盖所有边的权重 $w_{ij}$​，并且总顶标和最小。
### 2.2.2 算法流程
- **初始化**
    - 给二分图两侧分别编号：左集 U右集 V
    - 初始化顶点标号 $lx[i]=\max_j w[i][j]，ly[j]=0$
    - 初始匹配为空。
- **为每个左节点寻找匹配**
    - 对每个$u_i$​，尝试在相等子图（满足 $lx[i]+ly[j]=w[i][j]$ 的边）中找到一条增广路。
    - 若找到，则加入匹配。
    - 若找不到，说明相等子图不足，需要调整标号。
- **调整顶点标号**
    - 令$S$是 BFS/DFS 搜索到的左节点集合，$T$ 是对应的右节点集合。
    - 计算 $δ=\min⁡_{i\in S,j\notin T}\{lx[i]+ly[j]−w[i][j]\}$
    - 更新：
        $lx[i] -=  \delta,i\in S$ ，至少一条新的边加入相等子图
        $ly[j] += \delta, j\in T$，维持现有边的平衡，不影响已有可行匹配
    - 这样至少有一条新的边进入相等子图，继续寻找增广路。
- **重复上述步骤**
    - 直到所有左节点都有匹配，得到最大权完美匹配


# 3 多目标数据关联
多目标数据关联在工程里经常是非完美一一匹配：会有“漏检/消失”的轨迹（未被任何检测匹配），也会有“新生”的检测（未匹配任何轨迹）。另外实际工程中往往最小化代价匹配。
## 3.1 算法流程
1. 矩阵扩展（Dummy 节点/虚拟匹配）
	- 原始代价矩阵是 `T × D`（轨迹数 × 检测数）。
	- 为了允许“不匹配”，构造 **环形扩展** 的方阵，大小 `N = T + D`：
	┌──────────────┬────────────────────┐
	│  T×D 实际代价          │  T×T 轨迹→dummy列             │
	├──────────────┼────────────────────┤
	│ D×D 检测→dummy行 │  D×T dummy-dummy         │
	└──────────────┴────────────────────┘
	- 规则：
	    - 左上 T×D：真实 pair 的代价（小=好）。超出 gating 的设为 INF（禁止）。
	        
	    - 右上 T×T：只允许轨迹 _i_ 匹配它自己的 dummy 列 _D+i_：`cost = unmatched_track_cost`（对角），其余置 INF。
	        
	    - 左下 D×D：只允许检测 _j_ 被它自己的 dummy 行 _T+j_ 接住：`cost = unmatched_detection_cost`（对角），其余置 INF。
	        
	    - 右下 D×T：全部 INF（禁止 dummy-dummy 配对）。
        
	这样匈牙利/匈牙利-KM 仍是一一匹配，但**落到 dummy 上**就表示“未匹配”。
	
2. Gating 策略（先验过滤）
	- 在构造代价矩阵前，先用 **马氏距离 / IOU / 外观相似度** 判断两个目标是否可能匹配。
	- 超出阈值的元素直接置为 ∞，强制不匹配。
	
3. 得分阈值（后验过滤）
	- 运行 KM 得到匹配后，再检查对应代价/相似度是否超过阈值。
	- 如果过低，则强制认为“未匹配”。
	- 这样避免“勉强配对”的情况。
    
4. 生命周期管理
	- 给每个轨迹设置 **Missed Count**：连续几帧未匹配就删除。
	- 给检测设置 **Hit Count**：连续几帧被成功匹配才生成新轨迹。
## 3.2 dummy节点

### 3.2.1 为什么需要 Dummy 节点？

匈牙利算法要求：

> **所有行和列都必须被匹配一次**

但在 MOT 场景中：

- 轨迹可能 **没有对应检测**
    
- 检测可能 **没有对应轨迹**
    

为了把“未匹配”纳入同一个匹配框架，我们扩展矩阵，使得：

> **真实对象可以选择匹配 Dummy，而 Dummy 的匹配含义就是“不匹配”。**

---

### 3.2.2 Dummy 的语义

|位置|匹配含义|代价设置|
|---|---|---|
|**Track(i) → Detection(j)**|正常匹配|`cost(i,j)`（越小越好）|
|**Track(i) → Dummy(i)**|轨迹未匹配（消失 / 遮挡）|`unmatched_track_cost`|
|**Detection(j) → Dummy(j)**|检测未匹配（新目标）|`unmatched_detection_cost`|
|**Dummy ↔ Dummy**|无意义，禁止|`INF`|

Dummy 让匈牙利算法可以表达：

- “这个轨迹无人接续”
    
- “这个检测没有前身，应创建新轨迹”
    
即使 `T == D`，也能表示：

- “一个轨迹消失 + 一个新目标出现”
    
- “数量没变化但身份变化”
    
这是 `max(T,D)` 扩展所无法做到的。
