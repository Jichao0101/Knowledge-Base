# 1 多模态大模型系统笔记

本项目从**系统结构视角**整理多模态大模型

目标是回答一个底层问题：

> 一个完整的智能系统如何由语言模型逐步扩展形成？

这里的“扩展”不是简单增加参数量，而是沿着三个维度推进：

- 参数分布的优化
- 条件信息的扩展
- 环境闭环的接入

---

# 2 统一视角

生成模型的基本形式可以写为：

$$
P(x_t \mid x_{<t}, C)
$$

其中：

- $x_t$：当前生成 token
- $x_{<t}$：历史生成序列
- $C$：系统条件（context / condition）

这里的 $C$ 不是单一变量，而是一组系统级条件，可包含：

- prompt
- 对话历史
- 检索结果
- 工具输出
- 外部状态
- 环境观测

因此，系统能力可以抽象为：

$$
\text{能力} = f(\theta, C)
$$

其中：

- $\theta$：模型参数，决定内在策略分布
- $C$：条件信息，决定当前可利用的信息空间

由此可见，能力提升主要有两条正交路径：

1. **参数优化**：改变模型内部表示与策略分布  
2. **条件扩展**：改变推理时可见的信息与约束条件

---

# 3 系统层级

完整 AI 系统可以分为四层

## 3.1 模型层（Model）

模型层研究的是**概率模型本身**。

核心问题包括：

- token 预测
- 表示学习
- 序列建模
- 跨模态对齐

这一层关心的是：

$$
P(x_t \mid x_{<t})
$$

也就是：在不给额外系统结构的情况下，模型如何学习数据分布。

相关文档：

- [[LLM]]

---

## 3.2 训练层（Training）

训练层研究的是：**如何改变参数 $\theta$**。

核心方法包括：

- 预训练（Pretraining）
- 监督微调（SFT）
- 偏好优化（RLHF / DPO）
- 可验证奖励优化（Verifiable Reward）

这一层的本质不是改系统接口，而是通过优化目标改变模型分布：

$$
\theta \leftarrow \text{optimizer}(\text{loss})
$$

因此，训练层主要回答的是：

> 模型为什么会形成当前这种能力分布？

相关文档：

- [[LLM]]

---

## 3.3 系统层（System）  
  
系统层研究的是：**如何在模型外部引入状态、工具与控制流，使系统能够执行多步任务**。  
  
在这一层，模型不再只是被动地生成下一个 token，而是被组织进一个带状态的执行闭环中。  
  
底层生成模型仍可写为：  
  
$$  
P(x_t \mid x_{<t}, C)  
$$  
  
但系统的真正输出不再只看 token，而是看系统级动作：  
  
$$  
P(u_t \mid o_t, m_t, g, h_t)  
$$  
  
其中：  
  
- $u_t$：系统动作（system action）  
- $o_t$：当前观测  
- $m_t$：内部记忆  
- $g$：任务目标  
- $h_t$：历史轨迹  
  
典型系统动作包括：  
  
- Prompt 组织  
- RAG 检索  
- Tool Use  
- Memory Update  
- Planning  
- Runtime / Controller 调度  
  
因此，这层的关键变化是：  
  
> 系统从“单次 token 生成”走向“带状态的任务执行”。  
  
相关文档：  
  
- [[Agent]]

---

## 3.4 具身层（Embodied）

当系统不再只操作信息，而是需要与真实或仿真环境交互时，问题进一步变化。

此时系统输出的不再是系统动作 $u_t$，而是直接作用于环境的动作：

$$
P(a_t \mid s_t, g, h_t)
$$

其中：

- $a_t$：环境动作（embodied action）
- $s_t$：当前环境状态
- $g$：任务目标
- $h_t$：历史轨迹

典型结构为：

`perception → world model → policy → action`

这里的关键区别在于：

- 系统层的动作主要改变信息流与内部状态
- 具身层的动作直接改变外部环境状态

因此，系统从“任务执行器”进一步演化为“闭环决策与控制系统”。

- [[VLA]]

# 4 从 LLM 到 Agent

语言模型的基本能力是生成序列：

$$
P(x_t \mid x_{<t}, C)
$$

也就是：给定上下文，生成最可能的文本或符号序列。

但 Agent 的目标不是只生成文本，而是基于目标和状态持续选择系统动作：

$$
P(u_t \mid o_t, m_t, g, h_t)
$$

其中：

- $u_t$：系统动作
- $o_t$：当前观测
- $m_t$：内部记忆
- $g$：目标
- $h_t$：历史过程

因此，Agent 相比 LLM 增加的不是一点 prompt 技巧，而是完整的系统能力：

- Memory
- Tools
- Planning
- Runtime

可以抽象写为：

`LLM + Memory + Tools + Planning + Runtime = Agent`

这里最关键的变化是：

> 系统从“生成回答”变成“维护状态并选择系统动作”。

---

# 5 从 Agent 到具身智能

Agent 已经具备任务分解、工具调用和状态维护能力。  
但只要它的主要外部接口仍然是文本与软件工具，它就仍属于“数字环境中的任务执行系统”。

当系统进入物理环境后，还需要补上三类能力：

- 感知（Perception）
- 世界模型（World Model）
- 行为策略（Policy）

系统结构变为：

```text
perception
↓
world model
↓
policy
↓
control
```

此时，系统必须处理的问题包括：

- 部分可观测环境
    
- 状态随动作连续演化
    
- 长时域因果影响
    
- 真实执行成本与安全约束
    

因此，具身智能不是普通 Agent 的简单外接，而是：

> 在 Agent 之上加入环境状态建模与闭环控制。

---

# 6 系统演化路径

从当前技术路线看，系统大致沿如下路径演化：

Language Model  
↓  
Tool-Augmented LLM  
↓  
Agent  
↓  
Embodied Agent  
↓  
World Model + Policy

这条路径对应的，不只是产品形态变化，更是研究对象的变化：

- **Language Model**：学习语言分布
    
- **Tool-Augmented LLM**：引入外部能力接口
    
- **Agent**：形成多步执行闭环
    
- **Embodied Agent**：进入真实或仿真环境
    
- **World Model + Policy**：显式建模环境动态与行为策略
