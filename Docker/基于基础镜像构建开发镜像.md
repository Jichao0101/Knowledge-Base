

# 1 背景说明

地平线官方提供的基础镜像包含 SDK、交叉编译环境和运行时依赖，但不包含日常开发必需的工具（如 git、vim、调试工具等）。  
为了提升开发效率、保证环境一致性，基于官方镜像构建一个**开发镜像（dev image）**，用于日常开发与调试。

---

# 2 镜像体系结构说明

+---------------------------+
|  my-horizon-runtime:tag   |  ← 用于部署/交付，无调试工具
+---------------------------+
            ↑ COPY
+---------------------------+
|  my-horizon-dev:tag       |  ← 开发镜像（推荐使用）
|  含 git/vim/gdb/pytest 等 |
+---------------------------+
            ↑ FROM
+---------------------------+
| horizon/sdk:tag           |  ← 官方基础镜像（不可修改）
+---------------------------+


---

# 3 Dockerfile 示例（开发镜像）

将以下内容保存为 `Dockerfile.dev`
```
`# 基础镜像（由地平线提供）
FROM horizon/sdk:1.0
# 安装常用开发工具
RUN apt-get update && \
    DEBIAN_FRONTEND=noninteractive apt-get install -y --no-install-recommends \
        git \
        vim \
        less \
        tree \
        cmake \
        ninja-build \
        gdb \
        strace \
        net-tools \
        iproute2 \
        htop \
        openssh-client \
        ca-certificates \
        python3-pip \
    && rm -rf /var/lib/apt/lists/*

# 安装常用 Python 包
RUN pip3 install --no-cache-dir \
        numpy \
        pytest \
        ipython

# 可选：创建普通开发用户，避免直接使用 root
RUN useradd -ms /bin/bash dev
USER dev
WORKDIR /home/dev/workspace
```
`
> 如需增加其他工具（clang-format、ccache、debug-tools 等），在 `apt-get install` 后追加即可。

---

# 4 构建开发镜像

`docker build -f Dockerfile.dev -t my-horizon-dev:1.0 .`

构建完成后可通过以下命令确认镜像存在：

`docker images | grep my-horizon-dev`

---

# 5 启动开发容器

```
docker run -it \
    --name horizon-dev \
    -v /path/to/your/project:/workspace \
    my-horizon-dev:1.0 \
    /bin/bash

```

常用参数说明：

|参数|说明|
|---|---|
|`-it`|交互式启动|
|`--name`|容器命名，便于管理|
|`-v`|挂载本地代码目录到容器|
|`/workspace`|对应 Dockerfile 中 WORKDIR|

---

# 6 容器操作命令参考

启动已存在的容器：

```
docker start horizon-dev 
docker exec -it horizon-dev /bin/bash
```

删除容器：

`docker rm -f horizon-dev`

删除镜像：

`docker rmi my-horizon-dev:1.0`

---

# 7 开发镜像 vs 运行镜像区别说明

|项|开发镜像 dev image|运行镜像 runtime image|
|---|---|---|
|包含构建工具|✓|✗|
|包含调试工具|✓|可选 ✓/✗|
|包含 SDK|✓|✓|
|适用场景|日常开发/联调|部署/上车运行|
|镜像大小|较大|更小|

---

# 8 建议规范

- 基础镜像 **不得直接修改**，避免环境不可控
    
- 所有开发环境变更必须通过 **Dockerfile 管理**
    
- 镜像版本需跟随 SDK 版本一致管理，如：
    
    `my-horizon-dev:1.0 → 对应 horizon/sdk:1.0`
    

---

# 9 总结

- Dockerfile 用于构建可复现的开发环境
    
- `FROM` 指定基础镜像，`docker build` 构建镜像，`docker run` 启动容器
    
- 通过构建独立开发镜像，避免污染官方镜像，提高协作一致性
