# 1 ARM Cortex

在 ARM Cortex 体系中，**Profile 与架构版本是两个正交的约束维度**，共同定义一类处理器的架构规范，而微架构是其具体实现。

> **架构规范（Architecture specification）**  
> 由 **架构版本（ARMv7 / ARMv8 / ARMv9）** 与 **Profile（A / R / M）** 共同确定，  
> 定义软件可见的行为规则，包括 ISA、寄存器、异常模型、内存模型与调试机制。

> **Profile（A / R / M）**  
> 描述处理器的目标应用场景与能力取舍（通用计算 / 实时 / 微控制），  
> 用于约束在同一架构版本下**允许或禁止的特性子集**。

> **微架构（Microarchitecture）**  
> 是对某一 _（架构版本 + Profile）_ 组合的具体硬件实现
> 
```
                 Profile 维度
           ┌─────────┬─────────┬─────────┐
           │    A    │    R    │    M    │
┌──────────┼─────────┼─────────┼─────────┤
│ ARMv7    │ ARMv7-A │ ARMv7-R │ ARMv7-M │
│          │         │         │         │
│ ARMv8    │ ARMv8-A │ ARMv8-R │ ARMv8-M │
│          │         │         │         │
│ ARMv9    │ ARMv9-A │ ARMv9-R │   —     │
└──────────┴─────────┴─────────┴─────────┘
           ↑
      架构版本维度
```

# 2 编程模型
## 2.1 浮点运算单元

以  AArch64 的 Cortex-A*为例：

- 有 **32 个向量寄存器**：`v0, v1, ..., v31`
    
- **每个寄存器宽度 = 128 bit = 16 字节**
    
- 这 32 个寄存器同时承担两个角色：
    
    1. **标量浮点寄存器**（比如 double、float 的运算）
        
    2. **NEON SIMD 向量寄存器**（一次处理多元素）

### 2.1.1 浮点异常
浮点异常不会直接导致程序流程的改变，只是置位状态寄存器中的粘滞状态，可以将这些信号连接到处理器的中断输入端，来触发相应的中断

# 3 内存模型
## 3.1 对齐行为

Cortex-A支持未对齐访问，未对齐访问通常会触发底层 load/store 被拆分成两次访存，并可能跨 cache line，导致性能下降

使用普通C++，使用框架(PyTorch/TF/OpenCV/Eigen)时不需要对齐

### 3.1.1 手动对齐
- 写 NEON intrinsics / 手写算子
    
- 使用 DMA / Zero-Copy / 专有硬件加速器
    
- 做高性能卷积 / GEMM / 点云核

则建议对齐到：

- **16 字节**（SIMD 边界）
    
- **64 字节**（cache line 边界）

## 3.2 互斥
应用层互斥构建在底层原子原语之上，但应用层提供更多语义（优先级继承、阻塞等待）
### 3.2.1 应用层的 mutex：

- pthread_mutex
    
- std::mutex
    
- spinlock
    
- lock_guard
    

### 3.2.2 底层依赖：

- ARM 原子指令（LDXR/STXR 或原子加减）
    
- 内存屏障（DMB/DSB）保证顺序
    
- 内核调度器管理等待队列

## 3.3 缓存
### 3.3.1 局部性原则
#### 3.3.1.1 空间局部性
对一个位置的访问很可能随后会访问相邻的位置，例如顺序执行指令或数据结构的使用
```
for (int i=0; i<N; ++i)
    y[i] = a[i] + b[i];

```
当访问 `a[0]`，CPU 实际会取 **整个 64B cache line**。如果访问模式是规律的（stride=1）：
- 预取器会继续提前加载下一条 cache line `a[16]~a[31]`
- 程序到达时，数据已经在 L1/L2 里
**结论**：顺序访问能让预取器自动提前加载下一批数据，大幅减少 DRAM 访问成本
#### 3.3.1.2 时间局部性
短时间内，对内存区域的访问可能会重复发生，例如执行代码循环

在算法中反复使用同一块数据，例如：

- 卷积核 weight 在滑窗时多次复用

- 推理过程中某些 buffer 多次参与运算

### 3.3.2 缓存命中

对缓存中已有信息的访问称为缓存命中，其他访问称为缓存未命中。通常缓存是自我管理的，更新会自动进行。如果访问缓存命中，则立即进行访问；否则分配一个位置并从内存加载到缓存行

### 3.3.3 缓存一致性问题

1. CPU 与 DMA/外设共享 Buffer
	- CPU 写在 cache 里，但没写回 DRAM
	- DMA 读到的是旧值
	需要 `clean cache`

2. DMA 写回内存，但 CPU cache 仍旧
	- CPU 使用过期数据
	需要 `invalidate cache`

**结论**：如果不用 DMA，不需要操心一致性；  一旦用 DMA，就必须考虑 cache clean / invalidate，否则数据会错
